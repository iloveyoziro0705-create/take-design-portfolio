## 【最重要】BtoB評価前の必須確認事項

BtoC LP（美容・スクール等）と評価軸が根本的に異なる。以下3点を分析前に必ず確認すること。

```
[BtoB評価前チェック]
① 意思決定構造の確認
   → 担当者（提案者）+ 決裁者（承認者）の2名以上が関与
   → 担当者がLPから「社内提案用の情報」を収集するフローを念頭に評価

② CVアクションのフェーズ確認
   → 初期検討: 資料ダウンロード（最低ハードル）
   → 中期検討: デモ依頼・デモ動画視聴（機能確認）
   → 最終検討: 無料トライアル開始（実際に使う）
   → 「お問い合わせ」1種のみ → CTA設計の最大失敗

③ SaaSサブカテゴリ確認
   → HR・人事系: 法改正対応（電帳法・インボイス）訴求が有効
   → CRM・MA: ROI数字（営業生産性○%UP）が最重要
   → プロジェクト管理: 学習コストの低さ・フリーミアム
   → 会計・経理: 法改正対応明示が必須
```

## SaaS/BtoB LP分析の核心原則

### 1. FV評価の最重要ポイント
- **「何ができるか + 導入後の成果数字」が3秒で伝わるか**
  - 良い例: 「業務工数を40%削減。チームの生産性を変える。」+ UIスクリーンショット
  - 悪い例: ロゴ+英語コンセプトコピー「Empower Your Business.」のみ（何ができるか不明）
- **FV直下に「導入企業ロゴ一覧」**があるか（上場企業・有名企業含む = 社会的証明の最前線）
- **ツールUIのスクリーンショット or デモ動画**がFV内にあるか

### 2. 機能・ソリューション訴求の評価基準
```
機能訴求の評価軸:
× 悪い: 「リアルタイムダッシュボード機能搭載」（技術用語の羅列）
○ 良い: 「現場の進捗をリアルタイムで把握できる」（ユーザー視点の便益説明）

競合比較表の有無:
× なし: ユーザーが競合サイトに離脱して比較する
○ あり: 自社優位性をコントロールして訴求できる（= 離脱防止）
```

### 3. CTA設計の評価基準（BtoB3段階CTA）
```
検討フェーズ別CTAが揃っているか:
□ 初期検討層: 「資料ダウンロード」「事例PDF受け取り」（最低ハードル）
□ 中期検討層: 「デモを見る」「デモ依頼」（機能確認フェーズ）
□ 最終検討層: 「無料トライアル開始」「クレジットカード不要で試す」
```
- BtoCの「今すぐ予約」のような即決誘導CTAはBtoBでは逆効果
- 「担当者が社内提案しやすい」資料DLのCTAが最重要

### 4. 信頼性評価の三点セット（BtoB版）
```
導入事例 = 企業名（実名）+ 業種・規模 + 成果数字（「工数40%削減」等）
```
- 三点すべて揃っていれば信頼性+3点以上
- 「A社」「某製造業」等の匿名事例 → 信頼性ほぼゼロ

**BtoB特有の信頼性指標:**
- **導入企業数・ロゴ** = 上場企業・有名企業ロゴがあると最強の社会的証明
- **継続利用率○%** = 「使い続けている人が多い」という品質の間接証明（BtoC版継続率と同等）
- **セキュリティ認証バッジ** = ISO27001・SOC2・プライバシーマーク等（情報漏洩不安の解消）

### 5. 価格プラン評価
```
BtoBで価格を隠すと稟議・予算申請ができない → CVが絶対に発生しない
□ 3段階プラン比較表があるか（Starter / Pro / Enterprise）
□ 年払い割引の明示があるか
□ 無料トライアルの条件（CC不要・○日間）が明示されているか
□ 「詳細はお問い合わせ」のみは価格設計の最大失敗（-10点以上）
```

### 6. BtoB特有FAQ（必須3点）
```
以下3点のFAQ未設置は必ず減点:
□「セキュリティは大丈夫ですか？（情報漏洩リスク）」
□「既存ツールと連携できますか？（Slack/kintone/Salesforce等）」
□「導入後のサポート体制はどうなっていますか？」
```

## スコアリング評価軸（100点満点 / BtoB/SaaS専用基準）

**⚠️ BtoC基準（美容・スクール等）と採点軸・配点が異なる。必ずこの基準を使うこと。**

| 採点軸 | 配点 | BtoB特有の評価ポイント |
|-------|------|----------------------|
| FV | /20 | 「何ができるか+成果数字」が3秒で伝わるか / ツールUI表示があるか |
| 機能・ソリューション訴求 | /20 | 主要機能がユーザー視点で説明されているか / 競合比較表があるか |
| CTA設計 | /15 | 検討フェーズ別3段階CTA / 配置箇所数 / 資料DL+デモ+トライアルの3段 |
| 信頼性 | /15 | 導入企業数/ロゴ / 導入事例（数字付き）/ セキュリティ認証 |
| 価格プラン | /15 | 3段階比較表 / 年払い割引 / 無料トライアル条件明示 |
| 導線設計 | /10 | 課題共感→解決→実績→価格→CTAの流れ / FAQ充実度（BtoB必須3点） |
| モバイル最適化 | /5 | SaaSはPC閲覧が主流のため配点低め |

## 分析チェックリスト

### FV（/20）
```
□ 「何ができるか」がロゴなしでも3秒で伝わるか
□ 「導入後の成果数字」（工数○%削減・生産性○%UP等）が明示されているか
□ ツールUIのスクリーンショット or デモ動画があるか
□ FV直下（または内）に導入企業ロゴ一覧があるか
□ FVにCTA（資料DL or デモ依頼 or 無料トライアル）があるか
```

### 機能・ソリューション訴求（/20）
```
□ 主要機能が「ユーザー視点の便益」で説明されているか（技術用語の羅列でないか）
□ 「こんな課題はありませんか？」等の課題共感セクションがあるか
□ 課題→解決提示の流れが論理的か
□ 競合比較表（自社 vs 競合A vs 競合B）があるか
□ 機能の詳細説明（スクショ付き）があるか
```

### CTA設計（/15）
```
□ 「資料ダウンロード」CTAがあるか（初期検討層向け）
□ 「デモを見る / デモ依頼」CTAがあるか（中期検討層向け）
□ 「無料トライアル開始」CTAがあるか（最終検討層向け）
□ CTAが3箇所以上（FV・中盤・最終）に配置されているか
□ 「クレジットカード不要」「○日間無料」等の安心文言があるか
□ 「お問い合わせ」のみの一本化になっていないか
```

### 信頼性（/15）
```
□ 導入企業ロゴが掲載されているか（有名企業・上場企業含む）
□ 導入事例: 企業名+業種・規模+成果数字の三点セットがあるか
□ 導入社数・ユーザー数が数字で明示されているか
□ 継続利用率・解約率・NPS等の品質指標があるか
□ セキュリティ認証バッジ（ISO27001・SOC2・プライバシーマーク等）があるか
□ メディア掲載・受賞歴・第三者評価があるか
```

### 価格プラン（/15）
```
□ 料金プランがLP内に明示されているか（「詳細はお問い合わせ」のみは0点）
□ プランが3段階の比較表形式か（Starter/Pro/Enterprise等）
□ 年払い割引・月払い選択が明示されているか
□ おすすめプランがハイライト表示されているか
□ 無料トライアルの条件（CC不要・○日間）が明示されているか
□ Enterpriseプランに「カスタム相談」導線があるか
```

### 導線設計（/10）
```
□ 課題共感 → ソリューション提示 → 機能説明 → 事例 → 価格 → CTAの流れが自然か
□ FAQ: 「セキュリティは？」「既存ツール連携は？」「サポートは？」の3点があるか
□ 導入ステップ（申込→設定→運用開始）が視覚的に示されているか
□ 「担当者が社内提案しやすい」資料DL・事例PDF等のコンテンツがあるか
```

### モバイル最適化（/5）
```
□ スマホ表示でCTAボタンがタップしやすいか
□ 表・比較表がスマホでも読みやすいか（横スクロール or 縦並びへの対応）
□ テキストが読みやすいサイズか（16px以上）
```

## よくあるFAILパターン（SaaS/BtoB系）

| FAILパターン | 内容 | 減点目安 |
|-----------|------|---------|
| コンセプトFV型 | 「英語コピー+ビジュアルのみ」で「何ができるか」不明 | FV -10点 |
| UIスクショなし | ツールを使っているイメージが全くない | FV -5点 |
| 価格完全非開示 | 「詳細はお問い合わせ」のみ | 価格プラン -10点 |
| 匿名事例のみ | 「A社」「某製造業」等の企業名なし | 信頼性 -5点 |
| CTA一本化 | 「お問い合わせ」のみ（検討フェーズ別CTAなし） | CTA -8点 |
| 機能技術用語羅列 | エンジニア向け機能名のみ（担当者に伝わらない） | 機能訴求 -6点 |
| FAQ欠如 | BtoB必須3点（セキュリティ・連携・サポート）がない | 導線 -5点 |
| セキュリティ情報なし | セキュリティ認証・プライバシーポリシーへの誘導なし | 信頼性 -4点 |
| 競合比較表なし | ユーザーが自分で他社サイトを回って比較する設計 | 機能訴求 -3点 |
