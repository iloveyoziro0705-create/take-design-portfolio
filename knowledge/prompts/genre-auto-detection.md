# ジャンル自動振り分けルール

**用途**: 画像を受け取った際に自動でジャンルを判定し、適切な分析方針を適用するためのルール  
**参照元**: `governance.md` / `industry-patterns.md`  
**更新日**: 2026-06-20

---

## ルール概要

画像を受け取ったとき、Claude は以下の手順で**ジャンルを自動判定**し、分析方針を切り替える。
ユーザーに「このジャンルですか？」と確認せず、自力で判定して即座に分析を開始すること。

---

## ジャンル判定フロー

```
画像を受け取る
      ↓
以下の「視覚的シグナル」を確認（上から優先順位順）
      ↓
┌─────────────────────────────────────────────────┐
│ Step 1: テキストシグナル確認（最優先）              │
│ 「採用」「求人」「募集」「エントリー」→ 採用系     │
│ 「予約」「hair」「salon」「カット」「カラー」→ 美容系│
│ 「メニュー」「コース」「料理」「ご予約」→ 飲食系   │
│ 「クリニック」「医院」「受診」「症状」→ 医療系     │
│ 「物件」「間取り」「賃貸」「購入」→ 不動産系      │
│ 「スクール」「講座」「受講」「資格」→ 教育系      │
│ 「SaaS」「導入事例」「無料トライアル」「DX」→ BtoB系│
└─────────────────────────────────────────────────┘
      ↓ テキストシグナルで判定不能の場合
┌─────────────────────────────────────────────────┐
│ Step 2: 写真・ビジュアルシグナル確認               │
│ （各ジャンルの「写真に何が写っているか」を参照）   │
└─────────────────────────────────────────────────┘
      ↓
┌─────────────────────────────────────────────────┐
│ Step 3: CTAシグナル確認                            │
│ 「応募する / エントリー」→ 採用系                 │
│ 「ご予約 / LINE予約」→ 美容・飲食系               │
│ 「無料相談 / お問い合わせ」→ 金融・教育・医療系   │
│ 「無料トライアル / デモを見る / 資料DL」→ BtoB系  │
└─────────────────────────────────────────────────┘
      ↓
ジャンル確定 → 対応する分析方針に切り替え
```

**IND参照**: IND-001 / **プロンプト参照**: `beauty-lp-prompt.md`

| シグナル種別 | 判定キーワード・要素 |
|------------|-------------------|
| テキスト | 「hair」「salon」「サロン」「カット」「カラー」「パーマ」「ネイル」「エステ」「脱毛」「美容院」「美容室」「スタイル」「ヘア」|
| 写真 | 美容師・ハサミ・くし・ヘアスタイル・施術チェア・シャンプー台・鏡のあるカウンター |
| CTA | 「ご予約はこちら」「web予約」「LINE予約」「初回限定」「無料カウンセリング」|
| URL/タイトル | beauty・hair・salon・nails・eye・esthe 等 |

**分析方針の焦点**:
- FVにキャッチコピーがあるか / ブランドコヒーレンス
- 施術シーン写真 vs 内装写真の選択
- 予約CTAの配置・数
- 信頼性要素（製品提携・資格・ヘアドネーション等）
- Newsの更新状態（NewsStale確認）

### GEN-009: SaaS・BtoB系
**IND参照**: IND-008 / **プロンプト参照**: `saas-btob-lp-prompt.md`

| シグナル種別 | 判定キーワード・要素 |
|------------|-------------------|

| 写真 | ダッシュボードUI / アプリ画面スクリーンショット / オフィス内でPCを操作するビジネスパーソン / グラフ・数値レポート |
| CTA | 「無料トライアル」「デモを見る」「デモ依頼」「資料ダウンロード」「14日間無料」「クレジットカード不要」「まずは無料で試す」|
| 構成要素 | 導入企業ロゴ一覧 / 機能比較表 / 競合比較表 / 導入事例（成果数字付き）/ 料金プラン3段階表 / セキュリティ認証バッジ / FAQ（セキュリティ・連携・サポート） |

- 価格プランの透明性（「詳細はお問い合わせ」のみはBtoBで最大の失敗）
- BtoB特有FAQ3点: セキュリティ / 既存ツール連携 / サポート体制

## ジャンル判定後の分析方針切り替え

### 美容系（GEN-001）の分析チェックリスト
```
□ FVにキャッチコピーがあるか
□ ブランドコヒーレンス（テーマの一貫性）
□ FV写真: 施術シーン型 vs 内装型
□ 製品提携・認定の明示（イルミナカラー等）
□ 予約CTA: LINE/web予約の2択があるか
□ Newsの更新状態（最終更新日を確認）
□ ヘアドネーション等のCSR要素
□ スタッフ写真の有無
□ 価格表示の明確さ
```

### 採用系（GEN-002）の分析チェックリスト
**スコアリング基準の詳細 → `knowledge/lp/industry-patterns.md` の IND-004 参照**

| チェック項目 | 配点 | 確認ポイント |
|------------|------|-----------|
| FVで求職者の「悩み共感」ができているか | /15 | 悩み直撃型 / 理想提示型 / 仲間歓迎型 / 数字型のどれか |
| 給与・待遇が明示されているか（金額の具体性）| /15 | 「月収○万円〜」等の具体的数字があるか |
| 先輩の声: 顔写真+実名+入社前後の変化が3名以上 | /20 | 顔写真なし・名前なしは大減点 |
| 職場写真・施設の質と量 | /10 | 「どこで働くか」が視覚でわかるか |
| CTA: 3箇所以上+LINE誘導+低ハードル文言 | /15 | FV・中盤・最終の3点セット |
| FAQ+応募ステップ（視覚化）| /10 | 不安先取り / エントリー→内定を矢印で明示 |
| デザイン統一（業種配色）| /10 | 信頼感・親しみ・専門性がデザインで伝わるか |
| テクニカル（フォーム項目5以内・更新状態）| /5 | 項目過多による離脱防止 |

**判定宣言例**:
```
「採用LP（GEN-002）として分析します。スコアリングはIND-004基準（100点満点）で評価します。」
```

### 飲食系（GEN-003）の分析チェックリスト
```
□ FVに看板料理の写真があるか（品質チェック）
□ フリー素材を使用していないか
□ 価格帯の明示
□ 予約導線: 電話/web予約の導線が明確か
□ シェフ・スタッフの顔写真
□ 空間・雰囲気の訴求（照明・インテリア写真）
□ テイクアウト/デリバリー対応の明示（あれば）
```

### スクール系（GEN-006）の分析チェックリスト
**スコアリング基準の詳細 → `knowledge/lp/industry-patterns.md` の IND-006 参照**

| チェック項目 | 配点 | 確認ポイント |
|------------|------|-----------|
| FVで「受講後の成果」が数字で訴求されているか | /20 | 「3ヶ月で転職」「合格率〇%」等の数字があるか / ターゲット悩みへの共感があるか |
| 悩み→解決→実績→申込 の流れが自然か | /20 | セクション順序が論理的か / カリキュラム詳細が充実しているか |
| 無料体験CTAが3箇所以上・低摩擦文言か | /20 | FV・中盤・最終の3点セット / 「まず体験」等の文言 / スマホ固定フッター |
| 卒業生の声+実績数字の充実度 | /15 | 顔写真+職業+変化前後 / 合格率・就職率・受講者数 |
| 料金透明性+無料体験オファーの魅力度 | /15 | 月額換算表示 / 分割払い対応 / 特典・キャンペーン |
| スマホ申込の使いやすさ | /10 | フォーム項目5以内 / CTAボタンのタップ精度 |

**判定宣言例**:
```
「スクール系LP（GEN-006）として分析します。スコアリングはIND-006基準（100点満点）で評価します。」
```

### SaaS/BtoB系（GEN-009）の分析チェックリスト
**スコアリング基準の詳細 → `knowledge/lp/industry-patterns.md` の IND-008 参照**

**⚠️ BtoC基準（美容・スクール等）と採点軸・配点が根本的に異なる。必ずこの表を使うこと。**

| チェック項目 | 配点 | 確認ポイント |
|------------|------|-----------|
| FVで「何ができるか+成果数字」が3秒で伝わるか | /20 | ツールUIスクショ / 成果数字 / 導入企業ロゴ直下 |
| 主要機能がユーザー視点で説明・競合比較表があるか | /20 | 技術用語羅列でないか / 競合比較表の有無 |
| 検討フェーズ別CTA（資料DL→デモ→トライアル）があるか | /15 | 「お問い合わせ」一本化になっていないか |
| 導入企業数/ロゴ・事例（三点セット）・セキュリティ認証 | /15 | 匿名事例でないか / 認証バッジあるか |
| 料金プランの透明性（3段階比較表・無料トライアル条件）| /15 | 「詳細はお問い合わせ」のみは0点 |
| 課題共感→機能→事例→価格→CTA の流れ + FAQ3点 | /10 | BtoB必須FAQ: セキュリティ/連携/サポート |
| スマホ最適化（SaaSはPC主流のため低配点）| /5 | 比較表のスマホ表示が崩れないか |

**判定宣言例**:
```
「SaaS/BtoB系LP（GEN-009）として分析します。スコアリングはIND-008基準（BtoB専用100点満点）で評価します。」
```

## 判定に迷うケース

| ケース | 対処法 |
|-------|-------|
| 美容系 × 採用系が混在 | CVアクション（予約 or 応募）で判定。予約 → 美容集客LP、応募 → 採用LP |
| 医療系 × 美容系（美容クリニック） | 「美容クリニック」テキストがあれば医療系（法規制チェック必須）|
| テキストが日本語ゼロで英語のみ | ビジュアルシグナルで判定。CTAボタンの文言を優先 |
| 印刷物 vs ウェブスクショ区別不能 | A4縦サイズ・Canvaテンプレート感・QRコードがあれば印刷物として処理 |
| 複数業種が混在するポータル系 | メインCTAのジャンルで判定 |

---

## 判定結果の宣言ルール

分析開始前に必ず1行で判定結果を宣言する。

```
例:
「採用LP（GEN-002）として分析します。」
「美容サロン系LP（GEN-001）として分析します。」
「印刷フライヤー（GEN-008）として採点なしで知見抽出します。」
```
