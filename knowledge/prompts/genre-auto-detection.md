# ジャンル自動振り分けルール

**用途**: 画像を受け取った際に自動でジャンルを判定し、適切な分析方針を適用するためのルール  
**参照元**: `governance.md` / `industry-patterns.md`  
**更新日**: 2026-06-20

---

## ルール概要

画像を受け取ったとき、Claude は以下の手順で**ジャンルを自動判定**し、分析方針を切り替える。
ユーザーに「このジャンルですか？」と確認せず、自力で判定して即座に分析を開始すること。

---

## ジャンル判定フロー

```
画像を受け取る
      ↓
以下の「視覚的シグナル」を確認（上から優先順位順）
      ↓
┌─────────────────────────────────────────────────┐
│ Step 1: テキストシグナル確認（最優先）              │
│ 「採用」「求人」「募集」「エントリー」→ 採用系     │
│ 「予約」「hair」「salon」「カット」「カラー」→ 美容系│
│ 「メニュー」「コース」「料理」「ご予約」→ 飲食系   │
│ 「クリニック」「医院」「受診」「症状」→ 医療系     │
│ 「物件」「間取り」「賃貸」「購入」→ 不動産系      │
│ 「スクール」「講座」「受講」「資格」→ 教育系      │
└─────────────────────────────────────────────────┘
      ↓ テキストシグナルで判定不能の場合
┌─────────────────────────────────────────────────┐
│ Step 2: 写真・ビジュアルシグナル確認               │
│ （各ジャンルの「写真に何が写っているか」を参照）   │
└─────────────────────────────────────────────────┘
      ↓
┌─────────────────────────────────────────────────┐
│ Step 3: CTAシグナル確認                            │
│ 「応募する / エントリー」→ 採用系                 │
│ 「ご予約 / LINE予約」→ 美容・飲食系               │
│ 「無料相談 / お問い合わせ」→ 金融・教育・医療系   │
└─────────────────────────────────────────────────┘
      ↓
ジャンル確定 → 対応する分析方針に切り替え
```

## ジャンル別 視覚的シグナル一覧

### GEN-001: 美容・サロン系
**IND参照**: IND-001 / **プロンプト参照**: `beauty-lp-prompt.md`

| シグナル種別 | 判定キーワード・要素 |
|------------|-------------------|
| テキスト | 「hair」「salon」「サロン」「カット」「カラー」「パーマ」「ネイル」「エステ」「脱毛」「美容院」「美容室」「スタイル」「ヘア」|
| 写真 | 美容師・ハサミ・くし・ヘアスタイル・施術チェア・シャンプー台・鏡のあるカウンター |
| CTA | 「ご予約はこちら」「web予約」「LINE予約」「初回限定」「無料カウンセリング」|
| URL/タイトル | beauty・hair・salon・nails・eye・esthe 等 |

**分析方針の焦点**:
- FVにキャッチコピーがあるか / ブランドコヒーレンス
- 施術シーン写真 vs 内装写真の選択
- 予約CTAの配置・数
- 信頼性要素（製品提携・資格・ヘアドネーション等）
- Newsの更新状態（NewsStale確認）

### GEN-002: 採用・求人系
**IND参照**: IND-004 / **プロンプト参照**: `recruit-lp-prompt.md`

| シグナル種別 | 判定キーワード・要素 |
|------------|-------------------|
| テキスト | 「採用」「求人」「募集」「エントリー」「応募」「転職」「正社員」「パート」「アルバイト」「スタッフ募集」「一緒に働く」「仲間を募集」「JOIN US」「RECRUIT」「CAREER」|
| 写真 | スーツ/制服の社員・スタッフ、オフィス内観、作業現場、施設（病院・保育園・工場）、笑顔の社員集合写真 |
| CTA | 「応募する」「エントリーはこちら」「まずは話を聞く」「説明会に参加」「見学する」「LINEで質問」|
| 構成要素 | 先輩の声・1日の流れ・待遇/給与テーブル・応募ステップ・FAQ（経験不問？）|

**分析方針の焦点**:
- FVで「求職者の悩み共感」ができているか
- 給与・待遇の明示度
- 先輩の声（顔写真付き）の有無
- 1日の流れの具体性
- 応募フォームの項目数（5項目以上は離脱増）
- 応募ステップの明確さ
- FAQ（「経験がなくても大丈夫？」等の不安先取り）

## ジャンル判定後の分析方針切り替え

### 美容系（GEN-001）の分析チェックリスト
```
□ FVにキャッチコピーがあるか
□ ブランドコヒーレンス（テーマの一貫性）
□ FV写真: 施術シーン型 vs 内装型
□ 製品提携・認定の明示（イルミナカラー等）
□ 予約CTA: LINE/web予約の2択があるか
□ Newsの更新状態（最終更新日を確認）
□ ヘアドネーション等のCSR要素
□ スタッフ写真の有無
□ 価格表示の明確さ
```

### 採用系（GEN-002）の分析チェックリスト
```
□ FVで求職者の「悩み共感」ができているか
□ 給与・待遇が明示されているか（金額の具体性）
□ 先輩の声: 顔写真付きで3名以上あるか
□ 1日の流れ: 具体的な時間軸で記載されているか
□ 応募フォーム: 項目数は5以内か
□ 応募ステップ（エントリー→面接→内定）が明示されているか
□ FAQ: 「経験がなくても大丈夫？」等の不安先取りがあるか
□ 職場・スタッフ写真の量と質
□ CTA: 3箇所以上に設置されているか
□ ターゲット設定の明確さ（誰を採りたいか）
```

### 飲食系（GEN-003）の分析チェックリスト
```
□ FVに看板料理の写真があるか（品質チェック）
□ フリー素材を使用していないか
□ 価格帯の明示
□ 予約導線: 電話/web予約の導線が明確か
□ シェフ・スタッフの顔写真
□ 空間・雰囲気の訴求（照明・インテリア写真）
□ テイクアウト/デリバリー対応の明示（あれば）
```

## 判定に迷うケース

| ケース | 対処法 |
|-------|-------|
| 美容系 × 採用系が混在 | CVアクション（予約 or 応募）で判定。予約 → 美容集客LP、応募 → 採用LP |
| 医療系 × 美容系（美容クリニック） | 「美容クリニック」テキストがあれば医療系（法規制チェック必須）|
| テキストが日本語ゼロで英語のみ | ビジュアルシグナルで判定。CTAボタンの文言を優先 |
| 印刷物 vs ウェブスクショ区別不能 | A4縦サイズ・Canvaテンプレート感・QRコードがあれば印刷物として処理 |
| 複数業種が混在するポータル系 | メインCTAのジャンルで判定 |

## 判定結果の宣言ルール

分析開始前に必ず1行で判定結果を宣言する。

```
例:
「採用LP（GEN-002）として分析します。」
「美容サロン系LP（GEN-001）として分析します。」
「印刷フライヤー（GEN-008）として採点なしで知見抽出します。」
```
