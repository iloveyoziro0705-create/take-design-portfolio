# LP-Rank 評価ルール

**バージョン**: 1.0  
**作成日**: 2026-06-20  
**適用範囲**: 全LP分析・全業種

---

## 評価の基本方針

LPは「デザインの美しさ」ではなく **成果期待値（CVR）** で評価する。

```
評価する → CV構造 / 導線 / 信頼性 / CTA / オファー
評価しない → おしゃれさ / 配色センス / フォントの好み / 「なんとなく良い」
```

### 厳守事項

- 見た目の美しさだけで高評価にしない
- デザインが古くてもCV導線が優秀なら高評価にする
- デザインが綺麗でもCTAや導線が弱ければ低評価にする
- 業界大手だからという理由だけで高評価にしない
- 迷った場合は厳しめに採点する

## 採点表（合計100点）

### 1. FV（ファーストビュー）— 20点

| 採点項目 | 配点 | 採点基準 |
|---------|------|---------|
| キャッチコピーの強さ | 4 | 5=感情/悩み/数字で強烈に訴求 / 3=標準 / 1=弱い・抽象的 |
| 誰向けか明確か | 4 | 5=ターゲットが1文で特定できる / 3=推測できる / 1=不明 |
| ベネフィットが瞬時に伝わるか | 4 | 5=3秒で分かる / 3=読めば分かる / 1=読んでも分からない |
| 信頼要素があるか | 4 | 5=数字+顔写真+受賞などFV内に複数 / 3=1つある / 1=ない |
| CTAまで自然に視線誘導できるか | 4 | 5=F字/Z字で自然に到達 / 3=見つけられる / 1=見失う |

**注**: FVだけでCV判断する訪問者が最多。FVの評価は特に厳格に行う。

---

### 2. 導線設計 — 20点

| 採点項目 | 配点 | 採点基準 |
|---------|------|---------|
| セクション順が自然か | 4 | 5=AIDA/QUESTに沿った流れ / 3=概ね自然 / 1=バラバラ |
| 悩み→解決策→証拠→CTAの流れがあるか | 6 | 5=4ステップ全てある / 3=2〜3ある / 1=なし |
| ユーザーの不安を順番に解消しているか | 6 | 5=FAQと各セクションで先回り / 3=一部解消 / 1=未解消 |
| スクロールする理由が作れているか | 4 | 5=各セクションに続きを読む動機 / 3=一部ある / 1=ない |

**注**: 「悩み→解決策→証拠→CTA」の4ステップが揃うかどうかが最大の採点軸。

---

### 3. CTA設計 — 20点

| 採点項目 | 配点 | 採点基準 |
|---------|------|---------|
| CTAの設置位置が適切か | 4 | 5=FV/中盤/最終+固定フッターの4箇所以上 / 3=3箇所 / 1=1〜2箇所 |
| CTA文言が具体的か | 4 | 5=行動後のベネフィットを含む / 3=行動は分かる / 1=「こちら」等の汎用語 |
| ボタンが目立つか | 4 | 5=背景とのコントラスト最大・サイズ十分 / 3=見つけられる / 1=埋もれている |
| CTA直前に押す理由があるか | 4 | 5=実績/口コミ/希少性がCTAの直前にある / 3=近くにある / 1=ない |
| 複数CTAの役割が整理されているか | 4 | 5=LINE/電話/フォーム等の役割が明確 / 3=概ね整理 / 1=同じCTAの繰り返しのみ |

**注**: CTAが1〜2箇所しかないLPは設置位置で1点台にする。固定フッターCTAがないだけで-4点相当の影響を持つ。

---

### 4. 信頼性 — 15点

| 採点項目 | 配点 | 採点基準 |
|---------|------|---------|
| 実績・数値証拠 | 5 | 5=顧客数/年数/満足度等を複数の数字で明示 / 3=1つある / 1=ない |
| お客様の声・導入事例 | 5 | 5=名前+顔写真+具体的成果の口コミ3件以上 / 3=名前なしでも3件以上 / 1=なし or 1件以下 |
| 権威性（スタッフ/メディア/受賞/資格） | 5 | 5=複数の権威要素 / 3=1つある / 1=ない |

**注**: オープン前LP等で実績が取得できない場合は「取得不可（オープン前）」と記載し、代替要素（シェフ経歴・代表実績等）で部分加点する。

---

### 5. オファー — 15点

| 採点項目 | 配点 | 採点基準 |
|---------|------|---------|
| 行動する理由が明確か（無料/特典/診断/保証） | 7 | 5=「今すぐ行動すべき理由」が複数ある / 3=1つある / 1=ない |
| 希少性・緊急性・期限が設定されているか | 4 | 5=残り枠/期間限定/〇名限定 / 3=1つある / 1=ない |
| リスク低減要素があるか（返金保証/無料体験/相談のみOK） | 4 | 5=複数のリスク除去 / 3=1つある / 1=ない |

**注**: オファーはLPのCV率を最も直接的に左右する要素。無料体験・初回特典・返金保証がないLPは厳しく採点する。

---

### 6. モバイル最適化 — 10点

| 採点項目 | 配点 | 採点基準 |
|---------|------|---------|
| スマホで読みやすいか（フォントサイズ・行間） | 2 | 2=16px以上+行間1.7以上 / 1=概ね読める / 0=読みにくい |
| CTAが押しやすいか（タップ領域・固定フッター） | 2 | 2=44px以上+固定フッターあり / 1=どちらかある / 0=なし |
| 情報量が多すぎないか（1画面の密度） | 2 | 2=適切 / 1=やや多い / 0=過密 |
| セクションの切り替わりが分かりやすいか | 2 | 2=背景色/線で明確 / 1=概ね分かる / 0=分かりにくい |
| スクロール負荷が少ないか（不要な長さがない） | 2 | 2=適切な長さ / 1=やや長い / 0=明らかに長すぎる |

## LP-Rank 判定基準

| ランク | LP-Score | 説明 | ナレッジ登録 |
|--------|---------|------|------------|
| **S+** | 95〜100 | 業界最高クラス。永久保存レベル。今後の制作で最優先参照 | 全項目を積極登録 |
| **S** | 90〜94 | 非常に優秀。積極的に学習対象とする | 積極登録 |
| **A** | 80〜89 | 高品質。学習対象にする | 登録する |
| **B** | 70〜79 | 平均以上。部分的に参考にする | 優れた部分のみ抽出 |
| **C** | 60〜69 | 参考要素のみ抽出。辞典への登録は慎重に行う | 参考要素のみ保存 |
| **D** | 59以下 | 学習対象外。基本的にナレッジベースへ登録しない | 原則登録しない |

### Dランクの例外処理
Dランクでも「反面教師」として有用な場合は `learning-history/` に記録してよい。
その場合、必ず「Learning Decision: 反面教師として保存」と明記する。

---

## 学習対象ルール

```
S+ / S / A → 積極学習（辞典・inspirations/ に全要素を登録）
B          → 部分学習（優れた要素のみ抽出して登録）
C          → 参考保存（特定の参考要素のみ inspirations/ に保存）
D          → 学習対象外（原則 learning-history/ にも残さない）
              ※反面教師として有用な場合のみ記録可
```

## 出力フォーマット（必須）

LP分析時は必ず以下の形式で出力する。

```markdown
# LP-Rank Evaluation

## Score
総合点：XX / 100

## Breakdown
FV            : XX / 20
導線設計      : XX / 20
CTA設計       : XX / 20
信頼性        : XX / 15
オファー      : XX / 15
モバイル最適化 : XX / 10

## Rank
[S+ / S / A / B / C / D]

## Evaluation Reason
[なぜそのランクになったかを構造的に説明。感想・デザイン評価禁止。
 スコアを下げた主要因を3点以上、上げた主要因を3点以上記述する]

## Learning Decision
[積極学習 / 部分学習 / 参考保存 / 学習対象外 / 反面教師として保存]
```

## 採点で迷った場合のルール

1. **デザインが美しい → CV要因がない**: 低評価（CTAや導線が弱ければBより下）
2. **デザインが古い → CV構造が強い**: 高評価（構造・導線・CTAが揃えばA以上も可）
3. **オープン前LP（実績なし）**: 信頼性は「取得不可」として扱い、代替要素で最大8点
4. **業界大手LP**: ブランド力は採点外。構造・CTA・オファーのみで採点
5. **BtoB・高額商材（購買期間が長い案件）**: オファーに「無料相談・資料請求」があれば満点近くも可
6. **採点に迷う**: 厳しめの点数を採用する

---

## 過去採点の改訂ルール

採点ルールが更新された場合、過去のLP-Scoreは再採点する。
再採点時は `learning-history/` のファイルに「**[再採点済]**」と記載する。
