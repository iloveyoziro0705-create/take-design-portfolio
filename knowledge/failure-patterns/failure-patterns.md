## FAIL-001: CTA弱体化

### 名称
CTA弱体化

### 発生しやすい業種
全業種（特にデザイン重視案件で頻発）

### 問題内容
CTAの文言・配置数・視認性が不足しており、ユーザーが次に取るべき行動を判断できない状態。具体的には「文言が汎用的（こちら・詳しくはこちら）」「配置がFV+最終の2箇所のみ」「ボタンが背景と同化して視認できない」等の状態。

### なぜ失敗するのか
LP全体の情報・デザインが整っていても、ユーザーが行動に移るタイミングでCTAが視界に入らない・押す理由がない場合、行動に移らずページを閉じる。CTAはページ内の「行動トリガー」であり、出現頻度と具体性がCVRを直接決定する。

### CVに与える悪影響
- 中盤でページを読み終えた後の「どうすればいいか分からない」状態が発生
- ユーザーの関心がピークに達した瞬間にCTAが存在しないことによる行動機会の損失
- スマホでスクロール中にCTAが画面外にある状態が継続する

### 見分け方
- [ ] CTAがFV + 最終の2箇所のみである
- [ ] スマホ固定フッターCTAが存在しない
- [ ] CTAボタンの文言が「こちら」「詳しくはこちら」「お問い合わせ」等の汎用語
- [ ] CTAと背景のコントラスト比が3:1未満
- [ ] CTAの直前に「押す理由」（実績・口コミ・希少性）がない

### 修正方法
1. FV / セクション中盤（1〜2箇所）/ 最終CTAの最低3箇所にCTAを配置する
2. スマホ固定フッターCTAを追加する
3. CTA文言を「行動後のベネフィット」が伝わる具体的な表現に変更する（例:「無料でLINE相談する」「今すぐ予約する」）
4. CTAボタンのコントラスト比を4.5:1以上に調整する
5. CTAの直前セクションに口コミ・実績・希少性要素を配置する

### 代替パターン
- CTA-001: LINE誘導型（2択CTA）
- CTA-006: スマホ固定フッターCTA
- CTA-007: カウントダウンCTA（希少性）

### 関連Pattern
- Pattern-LP-001（美容サロン基本構成）のCTA配置図参照

### 参考事例
- `knowledge/learning-history/2026-06-20-restaurant-lp.md`（CTA設計: 12/20、設置位置: 1/4）
- `knowledge/learning-history/2026-06-20-millor-beauty-salon.md`（FVにCTAなし・スマホ固定フッターなし・LINE予約なし）
- `knowledge/learning-history/2026-06-20-sknow-beauty-salon.md`（FVにCTAなし・スマホ固定フッターなし確認）
- `knowledge/learning-history/2026-06-20-locco-beauty-salon.md`（FVにCTAなし — 部分該当。ConceptセクションとFooterにWEB予約+LINE QRコードあり。LP-Score: 56/100）

### 再発防止ルール
- [ ] 制作前にCTA配置図を作成し、最低3箇所（FV/中盤/最終）を事前確定する
- [ ] スマホ固定フッターCTAをデフォルト実装とする
- [ ] CTA文言に「汎用語」が含まれていないかチェックリストで確認する

## FAIL-002: 情報量過多

### 名称
情報量過多

### 発生しやすい業種
BtoB・採用・医療・不動産（情報量が多い業種で頻発）

### 問題内容
1つのセクション内にテキスト・画像・リストが詰め込まれ、1画面に表示される情報量がユーザーの処理能力を超えている状態。スマホ閲覧時に特に顕著。

### なぜ失敗するのか
人間の認知負荷（Cognitive Load）には上限があり、1画面に処理すべき情報が多すぎると「読むのが面倒」と判断して離脱する。特にスマホでは画面サイズが小さいため情報密度が上がりやすく、離脱率が高くなる。

### CVに与える悪影響
- スクロール率の低下（読まれないまま離脱）
- 重要な信頼要素・CTAが「情報の中に埋まって」視認されない
- ユーザーが「このサイトは難しい」と感じ、申込のハードルが上昇する

### 見分け方
- [ ] 1セクションに3つ以上の異なるトピックが存在する
- [ ] 本文テキストが1段落200文字以上で複数段落ある
- [ ] セクション内に3種類以上のビジュアル要素（写真・アイコン・リスト）が混在している
- [ ] スマホでセクションをスクロールするのに3回以上かかる

### 修正方法
1. 「1セクション1メッセージ」の原則を適用し、セクションを分割する
2. 本文の1段落を3〜5文（100〜150文字）以内に抑える
3. 箇条書き・表・アイコンで情報を構造化し、テキスト量を削減する
4. セクション間に十分な余白（パディング80〜120px）を確保して視覚的に分離する

### 代替パターン
- LAYOUT-001: min()コンテナによる適切なコンテンツ幅管理
- UI-002: フィーチャーカード（3列グリッド）による情報の分散

### 関連Pattern
- LAYINS-002: セクションパディングリズム

### 参考事例
- （今後の分析案件から追記予定）

### 再発防止ルール
- [ ] 各セクションのメインメッセージを1文で要約できるか確認する
- [ ] スマホプレビューで各セクションをチェックし、3スクロール超えのセクションを分割する

## FAIL-003: 信頼性不足

### 名称
信頼性不足

### 発生しやすい業種
美容・医療・金融・EC・採用（申込前に不安が生まれやすい業種）

### 問題内容
実績数値・お客様の声・運営者情報・資格・受賞歴等の信頼要素が不足または不在。「このサービスは本当に大丈夫か」というユーザーの不安を解消できない状態。

### なぜ失敗するのか
LPへのアクセス時点でユーザーは「知らないサービスに申し込む不安」を持っている。信頼要素はこの不安を段階的に除去する機能を持つ。信頼要素が不在だと、どんなに優れたオファーがあっても「本当に大丈夫か」という疑問が払拭されずCVに至らない。

### CVに与える悪影響
- 申込フォームへのアクセス後に「キャンセル」するユーザーの増加
- ページを読んだが申し込まないという「読了離脱」の増加
- 競合LPとの比較時に選ばれない

### 見分け方
- [ ] 実績数値（顧客数・年数・満足度）がページ内に存在しない
- [ ] お客様の声が0〜2件のみ、または名前・写真なし
- [ ] 運営者・スタッフの顔写真・プロフィールが不在
- [ ] 第三者認証・受賞・メディア掲載が存在しない
- [ ] LPのドメインが信頼できるか判断できる情報がない

### 修正方法
1. 実績数値をFVまたは直後のセクションに配置する（例:「累計500名以上」「満足度97%」）
2. お客様の声を名前・年代・顔写真・具体的成果の形式で3件以上掲載する
3. 運営者・スタッフのプロフィール写真と経歴を「権威性セクション」として追加する
4. 第三者認証（資格・受賞・メディア掲載）がある場合はバッジ形式でFVに配置する

### 代替パターン
- TRUST-001〜003: 各種信頼性UIパターン
- PROOF-001: 実績数字グリッド
- VOICE-001: 口コミカードUI

### 関連Pattern
- Pattern-LP-001（美容サロン）のSTEP 2「実績・数字」参照

### 参考事例
- `knowledge/learning-history/2026-06-20-restaurant-lp.md`（信頼性: 2/15）

### 再発防止ルール
- [ ] FV内または直後のセクションに実績数値を最低1つ配置する
- [ ] お客様の声を最低3件（名前+顔写真+具体的成果）確保する計画を立案する
- [ ] オープン前LP等で実績が取得不可の場合は代替要素（経歴・資格・権威）で補完する

## FAIL-004: FV訴求不明確

### 名称
FV訴求不明確

### 発生しやすい業種
全業種（デザイン優先・コピー後回しの案件で頻発）

### 問題内容
ファーストビュー内で「誰向けの」「何のサービスで」「どんなベネフィットがあるか」が明確に伝わらない状態。ユーザーが「自分ごと」として認識できずスクロールせず離脱する。

### なぜ失敗するのか
LPに訪問したユーザーは最初の3〜5秒でページを続けて読むかを判断する。FVでターゲット・ベネフィット・CTAの3要素が揃っていない場合、「自分には関係ない」と判断し離脱する。特にSNS広告・SEO流入は関連性の判断が最初の数秒で行われる。

### CVに与える悪影響
- ページのバウンス率が高く、広告費対効果が著しく低下する
- A/Bテストで「どこを改善すべきか」が特定しにくくなる
- 有望なターゲットにリーチできているのにCVにつながらない状態が継続する

### 見分け方
- [ ] H1コピーが抽象的・ブランド名のみ・告知のみ（例: 「GRAND OPEN」「〇〇サロン」）
- [ ] FV内に「誰向けか」を示すターゲット表現がない
- [ ] ベネフィット（なぜここを選ぶべきか）がFV内に記載されていない
- [ ] FVのCTAを押す理由がFV内で完結していない

### 修正方法
1. H1コピーを「ターゲット × ベネフィット」の公式で書き直す（例: 「初回体験で変化を実感できる美容鍼」）
2. サブコピーに具体的なベネフィットを1文で追加する
3. 信頼バッジ（実績数・評価点）をFV内に配置する
4. CTAボタンのテキストを「行動後に何が得られるか」を示す表現に変更する

### 代替パターン
- FV-001〜003: 各種FVパターン（コピー構造付き）
- FV-007: 悩みターゲット直接訴求型

### 参考事例
- `knowledge/learning-history/2026-06-20-restaurant-lp.md`（FV: 12/20、誰向けか: 2/4）
- `knowledge/learning-history/2026-06-20-millor-beauty-salon.md`（"働く、生きる、楽しむを１つに" — コンセプト型・文法ミス）
- `knowledge/learning-history/2026-06-20-sknow-beauty-salon.md`（"Hairstyle inspired by fashion." — 英語のみ・ターゲット不在）
- `knowledge/learning-history/2026-06-20-locco-beauty-salon.md`（**FAIL-004初回避** — H1「完全マンツーマンスタイルの美容室」で差別化訴求を明確化。美容室シリーズ3件中唯一の日本語コピー。FV: 12/20）
- `knowledge/learning-history/2026-06-20-bouquet-beauty-salon.md`（FVにテキストオーバーレイなし。良質コピー「髪のお悩みに真摯に応える 西大路のプライベートサロン」がConceptセクションに存在するが、FVに配置されていないためFV 4/20）

### 再発防止ルール
- [ ] FV制作前に「ターゲット × ベネフィット」フォーマットでH1コピーを確定する
- [ ] FV内に「ターゲット / ベネフィット / 信頼 / CTA」の4要素があるか確認する
- [ ] FVだけを見せて「誰向けのサービスか」が3秒で判断できるかチェックする

## FAIL-005: デザイン先行型LP

### 名称
デザイン先行型LP

### 発生しやすい業種
美容・ブランド系・高級商材（見た目の品質訴求が強い業種）

### 問題内容
ビジュアルデザインの完成度は高いが、CV構造（CTA配置・オファー・信頼要素・導線）が弱い状態。見た目は美しいがCVRが出ない。

### なぜ失敗するのか
デザインの美しさはユーザーの「好感度」を高めるが「行動する理由」を作らない。CVRを決定するのはデザイン品質ではなく「行動する理由（オファー）」「信頼できる証拠（信頼性）」「行動できる機会（CTA）」の3要素。これらが不在のままデザインだけが完成しても成果につながらない。

### CVに与える悪影響
- SNS等でシェアされ「おしゃれ」と評価されるがCVが発生しない
- 高いデザインコストが回収できないまま継続する
- クライアントが「なぜCV数が少ないか」を理解できず問題の発見が遅れる

### 見分け方
- [ ] LP-ScoreのFV・導線・CTA・オファーの合計が75点以下なのにデザインが美しい
- [ ] CTAが2箇所以下でオファーが不在
- [ ] お客様の声・実績数値が不在または弱い
- [ ] 導線が「コンセプト → ビジュアル → CTA」のみで悩み解消・証拠提示がない

### 修正方法
1. CV構造の優先度を「デザイン前」に確定する: CTA配置 → オファー設計 → 信頼要素配置 → デザイン
2. LP-Scoreの「FV / 導線 / CTA / 信頼性 / オファー」5項目を先に設計してからデザインを当てる
3. 既存のデザイン先行LPを修正する場合は、CTAとオファーの強化を最優先タスクとする

### 代替パターン
- Pattern-LP-XXX（各業種）の「セクション構成」通りに設計し直す
- FLOW-001〜003: 導線パターンで悩み→解決→証拠→CTAの流れを確保する

### 参考事例
- `knowledge/learning-history/2026-06-20-millor-beauty-salon.md`（LP-Score: 29/100、FAIL-001・003・004・005 全て該当。デザイン品質とCVRの非相関を最も明確に示す実例）

### 再発防止ルール
- [ ] 制作開始前に「オファー設計シート」でオファー・希少性・リスク低減の3要素を確定する
- [ ] デザイン着手前にLP-Scoreの想定値を算出し、80点以上の構成を確認してからデザインを開始する
- [ ] LP-Score採点でデザイン品質は評価項目に含まれないことを常に意識する

## FAIL-006: 外部CV完全依存型

### 名称
外部CV完全依存型

### 発生しやすい業種
美容室・飲食・小売・宿泊（Hotpepper Beauty / 食べログ / 楽天等の外部予約プラットフォーム依存型）

### 問題内容
自社サイト内の全CTA・予約機能を外部サービス（Hotpepper Beauty・食べログ・楽天等）に委託し、サイト内でのCV完結が一切できない設計。自社サイトがブランドカタログとしてのみ機能し、来客CVはプラットフォームに完全依存する。

### なぜ失敗するのか
外部プラットフォームは自社の競合サロン・店舗の広告を同一ページ内に表示するため、訪問者をHPB等へ誘導した時点で競合に奪われるリスクが発生する。また、プラットフォームの手数料改定・UI変更・解約時に全CV機能が即座に消失する構造的リスクを持つ。自社サイトからのCVR測定も外部依存のため困難。

### CVに与える悪影響
- 自社サイト訪問者がHPB等へ遷移後、競合サロンの広告に誘導されるCVの横流れ
- 外部プラットフォームの手数料改定・解約・仕様変更で全CV機能が消失するリスク
- LINE・電話等の低コストCVチャネルを自社管理できないまま高コスト外部依存が継続
- サイト改善によるCVR向上効果が外部プラットフォームの評価アルゴリズムに依存

### 見分け方
- [ ] 全予約・問い合わせリンクが外部URL（HPB・食べログ・楽天等）のみ
- [ ] 自社サイトに予約フォーム・LINE予約・電話CTA（誘導文あり）が存在しない
- [ ] スマホ固定フッターCTAが外部リンクのみ、またはスマホ固定CTA自体が存在しない
- [ ] フッターに電話番号はあるが「お電話でもご予約いただけます」等の誘導文がない

### 修正方法
1. **最低限**: 電話番号をスマホ固定フッターCTAとして配置し、「今すぐお電話でご予約」等の誘導文を追加
2. **推奨**: LINE公式アカウント予約をスマホ固定フッターCTAに追加する（美容室 / 飲食）
3. **理想**: 自社予約フォームまたは予約サービスの埋め込み（STORES予約・Airリザーブ等）を設置する
4. HPBへのリンクは維持しつつ、自社管理のCTAを並列配置して外部依存を分散させる

### 代替パターン
- CTA-006: スマホ固定フッターCTA（電話・LINE 2択型）
- CTA-001: LINE誘導型（2択CTA）

### 関連Pattern
- FAIL-001（CTA弱体化）: CTAが弱い・少ない問題。FAIL-006は「CTAが外部依存でゼロ」という根本的な別問題。

### 参考事例
- `knowledge/learning-history/2026-06-20-millor-beauty-salon.md`（全CTAがHotpepper Beauty外部リンク、サイト内CV完結不可、LP-Score: 29/100）
- `knowledge/learning-history/2026-06-20-locco-beauty-salon.md`（**FAIL-006初回避** — WEB予約（青）+ LINE QRコード（緑）の2択CTAを自社管理で設置。「24時間WEB予約中」バッジ付き。CTA: 16/20）

### 再発防止ルール
- [ ] 美容室・飲食等の案件では「HPB/食べログ依存型か自社管理型か」を要件定義時に確認する
- [ ] 自社サイトに電話番号スマホCTA（固定フッター）+ LINEの最低2チャネルを自社管理のCTAとして実装する
- [ ] HPBへのリンクは「クーポン確認」用として維持しつつ、メインCTAは自社管理とする方針を提案する

## FAIL-007: 更新停止コンテンツの前面配置

### 名称
更新停止コンテンツの前面配置

### 発生しやすい業種
美容室・飲食・企業HP・小売・サービス業（Wordpress等でブログ機能を設置したが運用を止めた案件）

### 問題内容
更新が止まったブログ・Newsセクション（最終更新が半年〜数年前）が、FV直後等のLP上位セクションに配置されている状態。サイト運営者の意図は「情報発信をしているサロン」という印象形成だが、古いコンテンツが上位に表示されると「現在も営業しているか」「サイトが放置されているか」という不安を訪問者に与える。

### なぜ失敗するのか
訪問者はLPをスキャンする際に日付・鮮度情報を無意識に確認する。「2019年」「3年前」等の日付がFV直後に出現すると、「このビジネスはまだ存在しているか」という根本的な疑念が発生する。この疑念は以降のConceptや信頼性セクションを読んでも完全に払拭されない。信頼を構築するために設置したコンテンツが、鮮度不足により逆に不信感のトリガーになる。

### CVに与える悪影響
- FV直後でバウンス率が上昇する（「このサイトは大丈夫か」と判断した段階での離脱）
- 以降のConceptセクション・信頼性セクションの効果が半減する
- 「最終更新2019年」等の表示はGoogleの評価上も更新停止サイトとして認識される可能性あり

### 見分け方
- [ ] News・Blogセクションの最終更新日が6ヶ月以上前
- [ ] NewsセクションがFV直後（セクション2〜3）に配置されている
- [ ] 最新記事の内容が「グランドオープンのお知らせ」等で数年前のもの
- [ ] ブログ記事数が2〜5件程度で止まっている（試しに設置して放置した状態）

### 修正方法
1. **即時対応**: 更新停止のNewsセクションをLPから非表示にする（HTMLコメントアウトまたはCSS `display: none`）
2. **推奨**: Newsセクションを削除し、代わりにInstagramフィード等の自動更新コンテンツに置き換える
3. **理想**: 3ヶ月に1回以上の更新頻度でブログ運営を再開する。更新できない場合はセクション自体を撤去する
4. **配置変更**: Newsセクションを維持する場合はフッター直上等の下位セクションに移動し、LP上位のCV構造に影響を与えない位置に変更する

### 代替パターン
- Instagramフィード埋め込み（自動更新・ビジュアル映え・SNS証明を兼ねる）
- 「お知らせ」を廃止してVOICE（口コミ）セクションをLP内に配置

### 関連Pattern
- FAIL-003（信頼性不足）: 信頼要素の不在。FAIL-007は「信頼要素が更新停止により逆効果になる」という別の問題。
- FAIL-005（デザイン先行型LP）: CV構造より見た目・機能優先。

### 再発防止ルール
- [ ] 既存サイトのリニューアル案件では、Newsセクションの最終更新日を確認し、6ヶ月以上の更新停止の場合はセクション削除を提案する
- [ ] 新規制作でブログ機能を設置する場合は「月1回更新」をクライアントに合意させてから設置する
- [ ] 更新できないと判明した場合は、Instagramフィード埋め込みへの変更を優先提案する
