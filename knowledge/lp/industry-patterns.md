# 業種別LP辞典

## 概要
業種ごとの最適LP構成・配色・心理学・CVポイントを体系化する。
「〇〇業種ならこの構成」を即座に提案できる状態を目指す。

**現在のステータス**: Phase 4（LP辞典150パターン到達後に本格育成）

---

## 業種管理ルール

- 業種ID: `IND-XXX`
- 各業種に「推奨パターン」「禁止パターン」を記録
- 実案件からの更新を優先

### Learning Reason（各業種パターン追加時に必須記録）

新規業種パターンを追加する際は、パターン末尾に必ず以下のセクションを記録すること。

```markdown
## Learning Reason
### 保存理由
[なぜこの業種パターンを独立して登録したか（業種特有の設計要件・発見経緯）]
### 再利用価値
[この業種パターンが有効な案件の範囲・規模]
### 特に優れている点
[業種固有のCV構造・ターゲット心理・禁止パターン回避の観点で優れている点]
### 他案件への応用先
[類似業種・周辺業種への展開可能性]
```

## IND-001: 美容・サロン系

**カテゴリ**: 美容院・ネイルサロン・エステ・脱毛・まつ毛

### 共通特性
- 意思決定者: 女性30〜50代（主力）
- 意思決定期間: 短〜中（1日〜1週間）
- 競合差別化軸: 技術力・雰囲気・価格・アクセス
- 主なCVアクション: LINE予約・電話予約・ネット予約

### 推奨構成（Pattern-LP-001参照）
```
FV（高級感） → 実績数字 → 特徴3点 → サービス → 
口コミ → Before/After → スタッフ → アクセス → FAQ → CTA
```

### 推奨カラーパレット
| ブランドポジション | カラー |
|----------------|--------|
| 高級・ラグジュアリー | `#faf7f3` + `#c8a56a` + `#3b2f2a` |
| ナチュラル・癒し | `#f9f5f0` + `#b5936c` + `#3d2b1f` |
| クール・モダン | `#f5f5f7` + `#8b7355` + `#1a1a1a` |
| ポップ・若者向け | `#fff0f3` + `#ff6b8a` + `#2d1b2e` |
| ナチュラル・ペールトーン | `#e8ece9` + `#f5ede8` + `#6b5c50`（ミントグリーン×クリーミーピンク × Liceriaフライヤーで確認） |
| スタイリッシュ・モノクロ+コーラル | `#ffffff` + `#1a1a1a` + `#e05050`（ミニマリスト高級感 × MADAサロンで確認） |
| エレガント・クリームゴールド | `#faf8f3` + `#4a3c30` + `#c8a560`（老舗・継承・格調系 × Hériterサロンで確認） |

### 禁止パターン
- ダークテーマ（信頼感・清潔感を損なう）
- テキスト過多のFV
- 料金表の曖昧表示
- **Hotpepper Beauty / 外部予約への完全委託**（FAIL-006該当 — サイト内CV機会がゼロになる）
- **来客CVページに採用セクションを大きく混在させる**（来客CV導線を阻害する）
- **更新停止ブログ/NewsセクションをFV直後に配置する**（FAIL-007該当 — 古い日付の記事が信頼性を逆効果にする）

### CV向上ポイント
1. LINE予約ボタンをFVに必ず配置
2. Before/Afterは"劇的な変化"を数枚用意
3. スタッフ写真は笑顔・白衣or制服着用
4. 口コミは「具体的な数字+地域+年代」付き
5. 固定フッターにLINEボタン（スマホ）
6. **STAFFセクション実装でLP-Score信頼性+3点（MILLORvssknow比較で確認）**
7. **PRICEセクション + 直後CTA 配置でLP-Score導線・オファー+4〜6点（同比較で確認）**
8. 「ヘアスタイル提案 → スタッフ紹介 → 価格確認 → 予約」の4ステップ導線が理想形
9. **FVコピーを日本語×差別化訴求型に変更するだけでFV +7〜8点（Locco「完全マンツーマンスタイルの美容室」12/20 vs 英語コピー系 4〜5/20）**
10. **WEB予約+LINE 2択CTA（FAIL-006回避構成）でCTA 16/20の最高点を実現（Locco検証）**
11. **初回限定10% OFFオファーバナー（Web予約限定条件付き）がオファー 5/15・CTA 14/20に貢献（bouquet検証）**
12. **スマホ固定フッター3択（CONTACT/ADDRESS/RESERVE）は役割分担明確でスマホCV確保に有効（bouquet検証）**
13. **VOICEセクションが別ページの場合、LP内の信頼性スコアが 1/15 まで低下する（bouquet検証）**

### 美容室LP 4件スコア比較（参考）

| サロン | FV | 導線 | CTA | 信頼性 | オファー | モバイル | 合計 | ランク |
|--------|-----|------|-----|--------|---------|---------|------|--------|
| MILLOR（武蔵小杉） | 4 | 7 | 5 | 2 | 2 | 9 | **29** | D |
| sknow（要確認） | 5 | 13 | 9 | 3 | 3 | 9 | **40** | D |
| bouquet（京都） | 4 | 12 | 14 | 4 | 5 | 8 | **47** | D |
| Locco（岡山・津山） | 12 | 13 | 16 | 3 | 5 | 7 | **56** | D |

**スコア差要因:**
- MILLOR→sknow +11点: STAFFセクション（信頼性+3）+ PRICEセクション（導線+6/CTA+4）
- sknow→bouquet +7点: 初回10% OFFオファー（+2）+ スマホ固定フッター3択（+5）
- bouquet→Locco +9点: 日本語差別化FVコピー（FV+8）/ bouquetはConceptにコピーあるがFVに不在
- Locco→C Rank最短: FVにCTA追加（+3）+ 口コミ3件（+4）= 63点見込み

**単一施策インパクトまとめ（4件から算出）:**
| 施策 | スコア増分 | 根拠 |
|------|-----------|------|
| 日本語差別化コピーをFVに配置 | +7〜8点（FV） | Locco vs MILLOR/sknow/bouquet比較 |
| STAFFセクション追加 | +3点（信頼性） | MILLOR vs sknow比較 |
| PRICEセクション + 直後CTA | +4〜6点（導線+CTA） | MILLOR vs sknow比較 |
| WEB予約+LINE 2択自社管理CTA | +5〜7点（CTA） | Locco vs sknow比較 |
| 初回10% OFFオファーバナー | +2〜3点（オファー） | bouquet vs sknow比較 |
| スマホ固定フッター3択 | +3〜5点（CTA+モバイル） | bouquet vs sknow比較 |

### Hotpepper Beauty（HPB）依存型サイトの注意事項
HPBは集客チャネルとして有効だが、自社サイトの全CTAをHPBに委託することには以下のリスクがある。
詳細: `failure-patterns.md` の FAIL-006 参照

| リスク | 内容 |
|--------|------|
| 競合誘導リスク | 訪問者がHPBへ遷移後、競合サロンの広告に誘導される |
| プラットフォームリスク | 手数料改定・解約・仕様変更で全CTA機能が消失 |
| CVR測定困難 | 自社サイト改善の効果がHPBのアルゴリズムに依存 |

**推奨対応**: HPBリンクは「クーポン確認用」として維持しつつ、電話番号とLINEを自社管理のCTAとして必ず設置する。

## IND-002: 飲食・レストラン系

**カテゴリ**: レストラン・カフェ・居酒屋・ラーメン・寿司・ベーカリー

### 共通特性
- 意思決定者: 全年齢層
- 意思決定期間: 超短（当日〜数日）
- 競合差別化軸: 料理の質・雰囲気・価格帯・アクセス
- 主なCVアクション: 電話予約・ネット予約・来店

### 推奨構成（Pattern-LP-002参照）
```
FV（料理ビジュアル） → コンセプト → 人気メニュー → 
空間・雰囲気 → アクセス → 予約CTA
```

### 視覚的優先順位
1. 料理写真の品質（最重要）
2. 空間・照明の雰囲気
3. シェフ/スタッフの顔

### 推奨カラーパレット（飲食系）
| ポジション | カラー | 印象 |
|----------|--------|------|
| イタリアン・洋食 | `#fffbf5` + `#c8a06a`（ウォームゴールド）+ `#3b2f2a` | 食欲・高級感 |
| カジュアルカフェ | `#fff8f0` + `#f97316`（テラコッタ）+ `#7c2d12` | 活発・温かみ |
| 和食・日本料理 | `#fafaf7` + `#5c4a1e`（抹茶ブラウン）+ `#1a1a1a` | 和の品格 |
| スパイシー・エスニック | `#f5e6c8` + `#c8400a`（バーントオレンジ）+ `#2a1a0a` | 食欲増進・本格感・深み（SPICE HOLIC確認）|
| 鉄板焼き・焼肉・炭火系 | `#1a0f00`（ディープブラック）+ `#c8650a`（アンバーオレンジ）+ `#8b5a00`（ダークゴールド）| 炭火・本格感・暗さの高揚感（HIROKI確認）|

**禁止配色**: 白×グレーのみの無彩色構成（清潔感は出るが食欲を刺激しない — A'unique分析で確認）

### MENUセクション 推奨レイアウト（A'unique分析で確認）
```
[カテゴリ名（英語大文字）] ← 左 or 右に大きな料理写真
                            ← 品名リスト（3〜4品）
                            ← 価格（必須）
                            ← MORE or 詳細へのリンク
→ 2カテゴリを左右交互に配置でスクロールリズムを生む
```

### CV向上ポイント（飲食系）
1. **料理写真は自社撮影を最優先** — ウォーターマーク付き素材は致命的信頼破壊（FAIL-008 — 更科分析で確認）
2. **老舗訴求「創業◯年」数字訴求** — 「創業1928年」等の歴史数字は飲食業界最強の信頼要素（更科分析で確認）
3. **EC展開バナー（商品配送訴求）** — 通販・商品配送を行う飲食店は中段にバナー配置で来店以外のCV機会を作る（更科分析で確認）
4. **欲求語コピー「今日も明日も食べたい」** — 日常的な食欲・リピート意向を引き出す（SPICE HOLIC分析で確認）
5. **専門性数字訴求「◯種類のスパイス」** — 具体的な数字で専門家ポジションを確立（SPICE HOLIC分析で確認）
6. **FVに看板料理写真** — 内装写真よりメインメニューの料理写真を最大化する
7. **産地×老舗 複合権威訴求「本場◯◯の◯年来の味」** — 産地ルーツ（本場）+ 老舗年数を組み合わせた2軸信頼訴求。特に地方料理の大都市展開店に有効（HIROKI分析で確認）
8. **多店舗展開信頼性訴求** — 2店舗以上の展開を地図付きで明示することで「選ばれ続けた人気の証明」になる（HIROKI分析で確認）

### 禁止パターン
- **ウォーターマーク付き素材の使用**（FAIL-008 — 著作権侵害疑惑で信頼が致命的破壊 — 更科分析で確認）
- フリー素材の料理写真（信頼性破壊）
- **予約ボタンの欠如**（CONTACT FORMのみでの運用 — A'unique分析で確認）
- 価格の非表示（飲食は「食べた後いくら？」が決断の最重要要素）
- FVに料理写真がない（内装写真のみ → 食欲を刺激しない）
- 複数業態の全サービスを1LPに詰め込む（訴求分散 — A'unique分析で確認）
- 白×グレーのみの無彩色構成（食欲を刺激しない）
- NewsStale Severity 3（プレースホルダーテキスト残存 — 更科分析で確認）

## IND-003: 医療・クリニック系

**カテゴリ**: 歯科・整形外科・皮膚科・美容クリニック・整骨院

### 共通特性
- 意思決定者: 全年齢（症状による）
- 意思決定期間: 短（痛みが強いほど即決）
- 競合差別化軸: 技術・痛みの少なさ・アクセス・待ち時間
- 主なCVアクション: ネット予約・電話予約

### 推奨構成
```
FV（安心感訴求） → こんな症状に → 当院の特徴 → 
治療の流れ → 実績・資格 → 患者の声 → 料金 → アクセス → 予約CTA
```

### 配色
- 歯科: `#f0f7ff` + `#2563eb` + `#1e3a5f`
- 整形・整骨: `#f0fff4` + `#16a34a` + `#14532d`
- 美容クリニック: `#faf7f3` + `#c8a56a` + `#3b2f2a`

### 法規制注意事項（必須）
- 効果の過剰表現禁止（医療広告ガイドライン）
- 「治る」「完治」等の表現不可
- 症例写真の使用条件（患者同意書必須）
- 費用の明示義務

## IND-004: 採用・求人系

**カテゴリ**: 保育士・介護士・看護師・建設・製造・IT・サービス業・小売・飲食スタッフ

### 共通特性
- 意思決定者: 求職者（転職意向あり / 復職希望 / 新卒・未経験）
- 意思決定期間: 中〜長（1週間〜1ヶ月）
- 競合差別化軸: 給与・働き方・職場環境・人間関係・成長機会・安定性
- 主なCVアクション: 応募フォーム・LINE問い合わせ・見学申込・説明会参加

### 推奨構成（Pattern-LP-003参照）
```
FV（悩み共感）
  → 求職者の悩み（共感 → 解決提示）
  → この職場の3つの特徴（数字入り）
  → 待遇・給与テーブル（具体的な金額必須）
  → 先輩スタッフの声（顔写真 + 名前 + 入社前後の変化）
  → 1日の仕事の流れ（時系列ステップ）
  → 施設・職場紹介写真
  → FAQ（「経験がなくても大丈夫？」等の不安先取り）
  → 応募の流れ（エントリー→面接→内定 を3ステップで明示）
  → 最終CTA（オファー付き）
```

### ターゲット別コピー差異
| ターゲット | FVコピー方向性 | 共感キーワード |
|-----------|-------------|-------------|
| 現役・転職希望 | 「今の職場のままでいいですか？」 | 「給与が低い」「人間関係」「評価されない」 |
| ブランク明け復職 | 「久しぶりの復職、不安なあなたへ」 | 「ブランク」「子育て後」「ブランクOK」 |
| 新卒・未経験 | 「未経験でも大丈夫。一から教えます」 | 「研修充実」「先輩が丁寧に」「資格取得支援」 |
| 経験者スカウト | 「あなたの経験を活かせる場所があります」 | 「即戦力」「キャリアアップ」「リーダー候補」 |
| W退職・ダブルワーク | 「週2日〜OK。自分のペースで働く」 | 「扶養内」「シフト自由」「副業OK」 |

### 推奨カラーパレット
| ポジション | カラー | 印象 |
|----------|--------|------|
| 信頼・安定（医療・介護・保育） | `#f0f7ff` + `#2563eb` + `#1e3a5f` | 安心・清潔・誠実 |
| フレンドリー・親しみ（小売・飲食） | `#fff8f0` + `#f97316` + `#7c2d12` | 明るい・活発・温かみ |
| 専門・キャリア（IT・建設・製造） | `#f5f5f7` + `#374151` + `#111827` | プロ・堅実・信頼 |
| ナチュラル・やさしい（保育・福祉） | `#f0fdf4` + `#16a34a` + `#14532d` | 癒し・安らぎ・自然 |

### 採用LP特有の心理設計

求職者は「美容室の予約」と異なり、**人生の大きな決断**をしようとしている。
意思決定のハードルが高いため、以下の心理的障壁を取り除く設計が必須。

| 心理的障壁 | 設計による解決策 |
|-----------|--------------|
| 「本当にいい職場か信用できない」 | 先輩スタッフの顔写真 + 実名 + 入社前後の変化（第三者の声）|
| 「自分に向いているか不安」 | ターゲット明示「〇〇な方に向いています」+ FAQ |
| 「応募したら断れない」 | 「まず話を聞くだけでもOK」「見学だけでも歓迎」という低ハードルCTA |
| 「給与・待遇が不透明」 | 具体的な月収額・時給を明示。「〇〇万円〜」でも必ず数字を出す |
| 「続けられるか心配」 | 定着率・平均勤続年数・研修制度の明示 |
| 「採用までの流れが不明」 | 応募ステップを3〜4段階で視覚化 |

### 採用LP CTAの鉄板文言（コピーライティング）

```
【低ハードル型（最初のCTA）】
- 「まずは話を聞いてみる（30秒でエントリー）」
- 「LINEで気軽に質問する」
- 「見学だけでもOK。お気軽にどうぞ」

【中ハードル型（途中のCTA）】
- 「一緒に働く仲間を探しています」
- 「あなたの経験を活かしてみませんか」

【最終CTA（オファー付き）】
- 「面接確約 / 今月中のエントリーで選考優遇」
- 「見学・体験入職受付中。まずはお話しましょう」
- 「応募フォームに進む（選考は2回まで / 最短1週間で内定）」
```

### 信頼性を高める数字の使い方

採用LPは「数字」が信頼性の核心。以下の数字を必ずヒアリングしてLP内に配置する。

```
✓ 定着率（「入社3年後定着率〇%」）
✓ 平均勤続年数
✓ 研修期間（「最初の〇週間はマンツーマン研修」）
✓ 年間有給取得率
✓ 産休・育休取得率（女性が多い業種で特に重要）
✓ スタッフ数・年齢層
✓ 月収モデル（「入社1年目モデル月収: 〇〇万円」）
✓ 資格取得支援実績
```

### FVコピーの構造パターン（採用系専用）

```
パターン A: 悩み直撃型（最高CVR）
「[今の職場の悩み] で悩んでいませんか？」
「[解決策 = 自社の強み] ここでは、[具体的な良いこと] ができます。」

パターン B: 理想の未来提示型
「[1〜2年後の理想の姿] を、ここで実現できます。」

パターン C: 仲間歓迎型
「[資格/経験/年齢/ブランク] は関係ありません。[自社の価値観] に共感できる方を歓迎します。」

パターン D: 数字インパクト型
「[定着率/年収/休暇日数] という数字が、私たちの自信です。」
```

### 禁止パターン
- 給与の非表示（求職者は必ず確認する。非表示は不信感の原因）
- 施設・職場写真がない構成（「どんな場所で働くか」が最重要視覚情報）
- 応募フォームの項目数過多（5項目以上で離脱増加。名前+電話+メールを基本に）
- 「未経験歓迎」「誰でも大丈夫」の根拠なし表示（具体的な研修内容で裏付けが必要）
- スタッフの声が「テキストのみ」（顔写真なしの声は信頼性が低い）
- 応募ボタンが1箇所のみ（最低3箇所：FV・中盤・最終CTAに配置）
- 会社概要だけが信頼性の根拠（「創業〇〇年」より「定着率〇%」が求職者に刺さる）

### CV向上ポイント
1. 先輩スタッフの声は顔写真 + 実名 + 「入社前の不安」→「入社後の変化」の流れで必須
2. 1日の流れは「〇:〇〇 出勤」から始まる時系列ステップで具体化する
3. FAQ（よくある質問）は「経験がなくても大丈夫ですか？」「ブランクがあっても大丈夫ですか？」を必ず含める
4. 応募ステップ（エントリー → 面接〇回 → 内定 → 入社）を矢印ステップで視覚化する
5. LINEへの誘導を最初のCTAにすると離脱が減る（フォームより心理ハードルが低い）
6. 「見学・体験入職」の案内を入れると応募意欲を高める（実際の職場を事前に見られる安心感）
7. 福利厚生はアイコン + 短文のグリッドで視覚化（テキストリストより読みやすい）

### 採用LP スコア評価の重点項目（美容系と異なる点）

| 評価軸 | 美容LP | 採用LP | 理由 |
|--------|--------|--------|------|
| FVコピー | 「日常価値/ブランド哲学」 | **「悩み共感 → 解決」** | 求職者の意思決定は感情+論理の両方が必要 |
| 信頼性の核 | 写真の雰囲気・技術感 | **先輩の声（顔写真+実名）** | 第三者の証言が最大の信頼性 |
| 価格表示 | あれば加点 | **必須（なければ大減点）** | 給与不明は応募離脱の最大原因 |
| CTA | 予約ボタン1〜2箇所で可 | **最低3箇所必須** | 意思決定に時間がかかるため複数接触が必要 |
| FAQ | あれば加点 | **必須（不安先取り）** | 「本当に大丈夫か」という不安を事前解消 |

## IND-005: 金融・資産運用系

**カテゴリ**: 投資・保険・不動産投資・FP相談・節税

### 共通特性
- 意思決定者: 30〜50代 会社員・経営者
- 意思決定期間: 長（1週間〜1ヶ月以上）
- 競合差別化軸: 実績・信頼性・担当者の質
- 主なCVアクション: 無料相談予約・セミナー申込

### 推奨構成（Pattern-LP-004参照）
```
FV（数字インパクト） → ターゲット条件 → サービス特徴 → 
実績数字 → 信頼性証明 → 顧客の声 → 相談の流れ → CTA
```

### 法規制注意事項（必須）
- 金融庁登録番号の表示
- リスク説明の表示義務
- 「必ず儲かる」等の表現禁止
- 顧客の声は実際の顧客から取得（捏造禁止）
