**バージョン**: 1.0  
**作成日**: 2026-06-24  
**管理ルール**: `knowledge/governance.md` の SST マップ参照（オファー設計パターンの定義はここに集約）

---

## このファイルの目的

LPで使用するオファー（特典・割引・インセンティブ）を体系化し、業種×目的に応じて最適なオファーを即選択できる状態を作ること。  
「なんとなく割引」ではなく、**オファーの種類・構造・期待効果を理解した上で設計する**ための辞典。

## OFFER-001: 初回割引オファー

**利用業種**: 美容・エステ・整体院・パーソナルジム・スクール・フィットネス  
**利用条件**: 実際に初回割引を実施できる価格設定があること

### 構造
```
通常価格: [通常価格]円
初回特別価格: [初回価格]円
割引率: [%]OFF
適用条件: 初回ご来店のみ / 全メニュー対象 / 当日追加料金なし
```

### 訴求ポイント設計
```
① 価格比較（通常 vs 初回）を視覚的に大きく表示
② 割引率（%OFF）を目立つバッジで表示
③ 「初回限定」という希少性を明示
④ 「全メニュー対象」等の条件の広さをアピール
⑤ 期間限定と組み合わせると緊急性が追加（TRIG-008と組み合わせ）
```

### 価格設定の目安
| 通常価格 | 推奨初回価格 | 割引率 | 期待CVR効果 |
|---------|-----------|-------|-----------|
| 8,000〜12,000円 | 3,000〜6,000円 | 50〜60% | 最高 |
| 15,000〜25,000円 | 5,000〜10,000円 | 50〜65% | 高 |
| 30,000円以上 | 9,800〜15,000円 | 50〜67% | 中高 |

### 実装HTML
```html
<div class="offer-box">
  <span class="offer-label">初回限定オファー</span>
  <div class="offer-prices">
    <div class="offer-old">
      <p class="label">通常価格</p>
      <span>12,000円</span>
    </div>
    <span class="arrow">→</span>
    <div class="offer-new">
      <p class="label">初回特別価格</p>
      <span class="price">¥5,980<small>円</small></span>
    </div>
    <div class="offer-badge">50%<br>OFF</div>
  </div>
  <p class="offer-note">税込 / 初回のみ / 全メニュー対象 / 追加料金なし</p>
</div>
```

### 注意点
- 「初回限定」を明記しないと2回目以降の価格誤認リスクあり
- 「追加料金なし」の明示で当日追加勧誘の不安を先制除去
- 割引率は本当に通常価格から計算すること（景品表示法対応）

### Learning Reason
**保存理由**: beauty-salon-lp（Luminos Skin）の STEP1-②（オファー強化）で「12,000円→5,980円（50%OFF）」のオファーをFVヒーロー要素として配置。初回割引は美容・エステ系LP最重要オファーパターンとして体系化。  
**再利用価値**: 美容・整体・スクール全般の初回体験型LPで必ず参照するオファー。  
**特に優れている点**: 通常価格との対比で「お得感」を視覚化できる。割引率(%OFF)バッジが即座の価値判断を促す。  
**他案件への応用先**: 全初回体験型サービス業種。

## OFFER-002: 無料相談オファー

**利用業種**: 士業（弁護士・税理士・社労士）・コンサル・不動産・採用支援・医療クリニック  
**利用条件**: 相談→成約の導線が成立する業種

### 構造
```
オファー: 初回相談○分 無料
提供内容: [相談で得られること]の具体的記載
予約方法: [LINE / フォーム / 電話] で[簡単に]予約
条件: [1社限定 / 先着○名 / 期間限定] など
```

### 心理設計
- 「無料」で試すことができるため、心理的ハードルが最低
- 「相談だけでもOK」という低ハードルCTAと組み合わせる
- 相談自体に価値を感じさせるコピーが重要（相談=無駄な時間 と思わせない）

### コピー例
```
「まずは無料でご相談ください。（30分 / オンライン可）」
「問題の整理から始めます。お気軽にご連絡を。」
「相談だけでもOK。決めなくても大丈夫です。」
```

### 期待効果
- 初期接点形成（リード獲得）
- 高価格サービスへの入口として機能
- 信頼関係の構築（会って話すことで成約率向上）

### 注意点
- 「無料」の範囲を明確にする（無料の後に何があるか）
- 相談内容の具体性を示す（「何について相談できるか」）

### Learning Reason
**保存理由**: 士業・コンサル・医療系LPで最も多く使われるオファーパターン。案件横断で再利用頻度が高いため保存。

## OFFER-003: 無料診断オファー

**利用業種**: 美容・スキンケア・ダイエット・SaaS（無料トライアル型）・保険・税務  
**利用条件**: 診断結果が個人にカスタマイズされた価値として提供できること

### 構造
```
オファー: [○○]診断 無料
所要時間: ○分程度
提供内容: 診断結果レポート / 個別アドバイス / タイプ診断
手順: ① [問診/アンケート] → ② [診断実施] → ③ [結果提供]
```

### 心理設計
- 「自分のことを知れる」という知的好奇心を刺激
- 診断結果を受け取ること自体が価値になる（結果 = リード育成コンテンツ）
- 診断完了後の「次のステップ（有料サービス）」への自然な導線

### コピー例
```
「あなたの肌タイプを無料で診断します。（2分でわかる）」
「あなたに合ったコースを無料でご提案。カウンセリング0円。」
「まず無料でツールを試してみてください。（14日間）」
```

### 期待効果
- 高いCTR（「無料で自分のことがわかる」訴求力）
- 診断完了率が高い（始めたら完了する行動特性）
- 有料転換率が高い（ニーズ確認済みのリードになる）

### 注意点
- 診断の質が低いと逆効果（「意味がない」と思われる）
- 診断後の成約提案タイミングが重要（早すぎると嫌われる）

### Learning Reason
**保存理由**: SaaS/美容/スキンケア系LP分析で高CVRパターンとして確認。特にスキンケア・美容系では「カウンセリング無料」がOFFER-001（初回割引）と並ぶ主要オファーパターン。

## OFFER-004: 資料請求特典オファー

**利用業種**: スクール・不動産・BtoB・保険・高額商材  
**利用条件**: 資料（カタログ・パンフ・提案書）が実際のCV価値になる業種

### 構造
```
オファー: 資料請求で[特典]をプレゼント
特典例: 
  - 限定PDF（〇〇チェックリスト / 成功事例集）
  - 割引クーポン同封
  - 先着○名プレゼント
  - 無料サンプル同封
手順: フォーム入力 → 即時 or 数日以内に郵送/メール
```

### 心理設計
- 「資料請求 = 買う気がある」という高温度リードの獲得
- 特典で「今すぐ請求する価値」を追加
- 資料がリード育成コンテンツになる

### コピー例
```
「資料請求はこちら（無料 / 特典PDF付き）」
「まずはパンフレットをご覧ください。プレゼントあり。」
「カタログ請求でオリジナルチェックリストをプレゼント。」
```

### 期待効果
- 高品質リードの安定獲得
- メールアドレス収集→ステップメール配信
- 成約まで時間がかかるサービスの育成手段

### 注意点
- 資料の品質が信頼性に直結する（手抜きNG）
- 「請求後すぐ電話がかかってくる」印象を与えないコピー選定

### Learning Reason
**保存理由**: スクール・不動産系LP分析で高CVRオファーパターンとして確認。CTA設計時の選択肢として参照する。

## OFFER-005: 期間限定特典オファー

**利用業種**: 全業種（キャンペーン型LPに適用）  
**利用条件**: 実際に期間が設定されていること（虚偽期限は景品表示法違反）

### 構造
```
オファー種別（組み合わせ自由）:
  A. 期間限定割引（○月○日まで〇%OFF）
  B. 期間限定特典（今月申込で〇〇をプレゼント）
  C. 先着限定（先着○名に〇〇特典）
  D. 数量限定（残り○席 / ○個）

緊急性を高める要素:
  ① カウントダウンタイマー（JavaScript）
  ② 残席数表示（「残り○席」）
  ③ 期限バッジ（「〇月〇日まで」）
```

### 心理設計（TRIG-005 損失回避 + TRIG-008 緊急性との組み合わせ）
- 「今やらなければ機会を失う」という損失回避を刺激
- 「残り少ない」という希少性が欲求を高める
- 具体的な期限・数字が信頼性を高める

### コピー例
```
「〇月〇日（〇）まで。この価格での初回体験は今月限り。」
「残り5名様分のみご案内しています。」
「今月中にお申込みいただいた方に限り、5,980円でご案内します。」
```

### カウントダウンタイマー実装
```javascript
// 期限日時の設定
const deadline = new Date('2026-07-31T23:59:59');
function updateTimer() {
  const now = new Date();
  const diff = deadline - now;
  const days = Math.floor(diff / (1000 * 60 * 60 * 24));
  const hours = Math.floor((diff % (1000 * 60 * 60 * 24)) / (1000 * 60 * 60));
  document.getElementById('countdown').textContent = `残り${days}日${hours}時間`;
}
setInterval(updateTimer, 60000);
updateTimer();
```

### 注意点
- 期限の虚偽設定は景品表示法違反（毎月延長はNG）
- 「期間限定」の連続使用は信頼性を損なう
- カウントダウンタイマーが実際に機能していること

### Learning Reason
**保存理由**: STEP1-②（オファー強化）で「期間限定特価」バッジを追加した際の設計根拠。全業種で使える汎用オファーパターンのため最重要カテゴリとして保存。

## OFFER-006: LINE登録特典オファー

**利用業種**: 美容・飲食・スクール・フィットネス・採用（LINE活用業種全般）  
**利用条件**: LINEを予約・問い合わせ導線として運用していること

### 構造
```
オファー: LINE友達追加で[特典]プレゼント
特典例:
  - 初回割引クーポン（○%OFF / ○○円OFF）
  - 限定コンテンツ（スキンケアガイドPDF等）
  - 優先予約権（満員でも先行案内）
  - 誕生日特典
手順: QRコード or ボタン → 友達追加 → 自動メッセージで特典配布
```

### 心理設計
- LINE登録のハードルは低い（メールより簡単）
- 特典がリード獲得の動機になる
- 登録後のリッチメニュー・ステップ配信でリード育成可能

### コピー例
```
「LINEで友達追加 → 初回割引クーポンをお送りします。」
「友達追加後、日時をLINEでお送りください。24時間受付。」
「LINE登録でスキンケアガイドPDFを無料プレゼント。」
```

### 期待効果
- 高いリード獲得率（LINEは返信率がメールの10倍以上）
- リピート誘導のしやすさ（LINEでメッセージ送信可能）
- 予約・問い合わせ対応の効率化

### 注意点
- LINE公式アカウントの運用体制が必要
- 特典の配布タイミングと方法を事前設計する
- LINE登録を「予約完了」と混同しないコピー

### Learning Reason
**保存理由**: beauty-salon-lp（Luminos Skin）ではLINE予約が主要CTA。CTA-001（LINE誘導型）と連動したオファー設計として保存。美容・飲食系LP全般で高頻度利用。

## OFFER-007: モニター募集オファー

**利用業種**: 新サービス・開店間もない店舗・新商品・新コース導入時  
**利用条件**: 正式リリース前 or 実績形成フェーズであること

### 構造
```
募集内容:
  - 枠数: [○名限定]
  - 価格: [通常価格の○%OFF / 無料 / 謝礼あり]
  - 条件: [SNS投稿 / アンケート回答 / 写真提供 / 口コミ協力]
  - 期間: [○月○日まで募集]
```

### 心理設計
- 「選ばれる」という希少性が欲求を高める
- 「モニター＝特別扱い」という付加価値
- 掲載実績・口コミを初期段階で獲得できる
- ユーザー生成コンテンツ（UGC）の獲得手段としても機能

### コピー例
```
「先着5名様限定でモニター様を募集しています。（通常の60%OFF）」
「開院記念のモニター施術にご協力いただける方を募集。」
「SNS投稿にご協力いただける方、特別価格でご案内します。」
```

### 期待効果
- 低リスクで実績・口コミを積める
- 初期CVRの向上（「モニター」という言葉の希少性）
- 口コミ・SNS拡散で自然な集客増加

### 注意点
- 「モニター」の条件は明確に記載する（後出しNG）
- 実績が少ない段階では信頼性の低さを補うオファーとして機能
- 実績が積まれたらOFFER-001に切り替える

### Learning Reason
**保存理由**: 新規開業サロン・新サービスLP設計で必要なオファーパターン。実績ゼロ時の信頼性構築手段として体系化。

## OFFER-008: 返金保証オファー

**利用業種**: スクール・高額商材・SaaS・オンラインサロン・コンサルティング  
**利用条件**: 返金対応が事業的に可能であること

### 構造
```
保証内容: [○日間 / ○回] 全額返金保証
条件: [条件を満たさない場合 / 理由を問わず / 〇〇の基準に達しない場合]
手順: [申請方法]（LINE / フォーム / メール）
```

### 心理設計
- 購買リスクをゼロに近づけることで行動障壁を最低化
- 「返金保証がある = 自信がある」という逆説的信頼形成
- 「最悪、返金できる」という合理的判断の後押し

### コピー例
```
「30日間全額返金保証。効果を感じなければ全額お返しします。」
「試して合わなければ返金。だから安心して始められます。」
「60日以内なら理由を問わず返金対応します。」
```

### 期待効果
- 高額商材のCVR最大化（リスク軽減 = 購買障壁除去）
- 「でも万が一は？」という最後の迷いを解消
- 実際の返金率は通常1〜5%程度（多くの人は試してから判断）

### 注意点
- 返金条件の細則を明確に記載する（「30日間」「全額」等の具体化）
- 「条件が複雑すぎる」保証は逆効果（シンプルに）
- 適用外条件を明記（消費者保護の観点）
- 資金繰りへの影響を事前試算する

### Learning Reason
**保存理由**: スクール・高額商材LP分析で、返金保証がCVR向上の強力なレバーになることを確認。特に価格が30,000円を超える商材では必須検討オファーとして体系化。
