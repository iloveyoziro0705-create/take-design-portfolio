## 最大の発見: 「仕事も遊びもプロフェッショナル」カルチャーステートメント

### ★新規パターン: スタッフ紹介の見出しを「会社文化の宣言」にする

通常の採用LPのスタッフ紹介見出しは「社員の声」「先輩インタビュー」等の機能説明型。
クラスメソッドは以下の見出しを採用:

```
「仕事も遊びもプロフェッショナル
  それがクラスメソッド社員の当たり前です」
```

この見出しの機能:

| 機能 | 内容 |
|-----|------|
| **カルチャー宣言** | 「プロフェッショナル = 当たり前」という文化レベルの高さを断言 |
| **自己選別（セルフスクリーニング）** | 「当たり前です」という表現で「これが普通の環境」を示す → 共感できる人だけが残る |
| **二面訴求** | 「仕事も遊びも」→ 仕事への熱意 + 生活の充実を両立している組織文化を示す |
| **ハードル設定** | 「高いレベルが求められる → でも報酬・環境も高い」という暗示 |

**「当たり前です」という断言の威力**:  
「〇〇を目指しています」「〇〇な職場を作りたい」等の「未来形・希望形」ではなく  
「**それがクラスメソッド社員の当たり前です**」という現在完了形・断言が  
「文化が既に完成している組織」という信頼性を生み出す。

→ IND-004 新パターン: **カルチャーステートメント型スタッフ紹介見出し**

## 「Developers.IO」コンテンツとの採用LP連動

### エンジニア採用特有の強力な差別化要素

クラスメソッドは「Developers.IO」という技術ブログ（月間1000万PV規模）を運営。
「働くメリット」詳細ページに「Developers.IO」のロゴ・コンテンツが組み込まれている。

**この仕組みの革新性**:

```
技術者が「Developers.IO」の記事を読む
        ↓
「記事のクオリティ = この会社のエンジニアが書いている」と認識
        ↓
「この会社のエンジニアはすごい」という技術力への信頼が形成される
        ↓
採用LPへのルートができる
```

採用LPと「採用前教育コンテンツ（技術ブログ）」を接続する設計は、  
**「LP到達前から信頼が形成されている」状態を作る**。

「百聞は一見に如かず → 百記事は千コピーに如かず」（技術者への信頼訴求としての記事）

→ GEN-002 新パターン: **オウンドメディア・技術ブログを採用LPに統合する設計**

## 勤務制度セクション: 「働き方」の早期訴求

IT/エンジニア採用で最重要の「働き方」を専用セクションで早期に訴求。

```
確認できた情報:
・在宅ワーク（フルリモート）
・変形労働時間制
```

**「フルリモート」訴求のIND-004での位置付け**:

採用LPの「給与・待遇開示」は月収額の明示が最重要だが、  
IT/エンジニア採用では「働き方（リモート可否・フレックス）」が給与と並ぶ優先項目。

IND-004の推奨数値（信頼性を高める数字）に「在宅勤務率」があるが、  
クラスメソッドは「フルリモート可」という条件そのものを前面に出している。

## 問題点

### 問題1: 給与の具体額が不在

フルリモートや働き方は訴求できているが、具体的な月収モデルが確認できない。  
エンジニア採用では「年収〇〇〜〇〇万円」の具体額が転職意欲に直結。

### 問題2: 入社前後の変化が詳細ページ送り

6名の社員インタビュー（顔写真+名前+職種）は優秀だが、  
「入社前の不安→入社後の変化」の詳細は「more→」リンク先のページにある。  
LP内（1ページ内）で3名分でも入社前後の変化を見せれば、スタッフの声スコアが+5〜7点向上する。

### 問題3: FVコピーの文言が確認しにくい

手書き風フォントは視覚的インパクトがあるが、  
画像解像度が低い状況では読めない。  
コピーの正確な文言が伝わらない場合、第一印象が弱まるリスクがある。

## 新規パターン（IND-004追記対象）

### パターン1: カルチャーステートメント型スタッフ紹介見出し

```
「仕事も遊びもプロフェッショナル、それがクラスメソッド社員の当たり前です」
```

単なる「社員インタビュー」「先輩の声」ではなく、スタッフ紹介セクションの見出しを  
**「会社文化の宣言（ステートメント）」**にすることで、  
文化に共感できる人だけを自然に選別するセルフスクリーニング効果が生まれる。

応用テンプレート:
```
「[仕事スタンス]。それが[会社名]の当たり前です。」
「[価値観] に共感できる人と、一緒に働きたい。」
「[文化の定義] — これが私たちの働き方の基準線です。」
```

### パターン2: オウンドメディア・技術ブログを採用LPに統合する設計

技術ブログ・SNS・Podcastなどの「採用前教育コンテンツ」を採用LPに接続することで、  
「LPに来る前から信頼が形成されている」状態を作る。

応用例:
- IT/エンジニア採用 × 技術ブログ（Developers.IO等）
- 飲食・料理人採用 × レシピブログ・シェフのInstagram
- 教育・コーチ採用 × YouTube・Podcast
- 採用ブランディング × 社員ブログ（日常発信）
