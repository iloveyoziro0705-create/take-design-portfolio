## ⑧ 参考パターン・知見抽出

### 参考パターン1（★最優先）: GEN-006初の「LP内料金テーブル」

**構造**:
```
Weekly プラン  ┃  Monthly プラン
¥〇〇〇/回    ┃  ¥〇〇,〇〇〇/月
〇回/週       ┃  〇回/月
（特徴説明）   ┃  （特徴説明）
```

**機能する理由**: 週単位・月単位でのコストを比較できることで「どちらが自分に合うか」という意思決定をLP内で完結させる。GEN-006-001〜007が全て「料金不明示」だったことを考えると、このテーブルの存在だけでオファー評価が11点（シリーズ最高）に跳ね上がった。

**重要教訓**: GEN-006全体で「料金をLP内で見せる」かどうかが、オファーの分水嶺になる。

### 参考パターン2: 講師5名プロフィールの充実による信頼性構築

**構造**:
```
[講師カード ×5]
顔写真 / 氏名 / 専門科目 / 経歴 / メッセージ
```

スクール系LPにおける「誰から学ぶか」の不安を解消する最良のコンテンツ。特に個別指導・コーチング型スクールでは「先生との相性」が申し込みを左右するため、講師一人一人の個性が伝わるプロフィール設計が重要。

### 参考パターン3: 「大学受験のその先へ」という未来視点FVコピー

一般的な受験塾が「〇〇大学合格！」という結果を訴求するのに対し、「その先へ」という未来（大学入学後・社会人）を示すコピーは差別化として機能する。ただし、成果数字（合格率）がないと「で、実際に合格できるの？」という疑問が残るため、未来訴求 + 合格実績の組み合わせが最も強い。

### 参考パターン4: FAIL-011のLP内確認

NEWSセクションがFV直後に配置されている可能性（確認）。GEN-004でも繰り返し確認したFAIL-011（お知らせファーストセクション問題）がスクール系でも発生していることを確認。CMSデフォルト問題として業種横断的な共通FAILパターンとして記録。

## Learning Reason

### 保存理由
GEN-006-008として、GEN-006シリーズ内で最も設計が充実した大学受験・就職面接専門塾LPを記録。特に「LP内料金テーブル」「講師5名プロフィール」「メディア掲載」の三点が揃った事例はスクール系LPの「B〜A設計に必要なもの」を示している。また大学受験塾という「資格合格型」サブカテゴリの初事例として記録。

### 再利用価値
- 大学受験塾・予備校・学習塾全般
- 個別指導・コーチング型スクール（週次/月次のプラン比較設計）
- 講師紹介が重要な専門スクール全般

### 特に優れている点
Weekly/Monthlyの2プラン比較テーブルは、ユーザーの「通えるペースに合わせて選べる」という柔軟性を示しながら料金不安を同時に解消する優れたオファー設計。個別指導型スクールの料金提示の参考モデルになる。

### 他案件への応用先
- 英会話スクール（週1・週2・週3プランの比較）
- プログラミング個別指導（週次・月次プラン）
- パーソナルトレーニング（回数券・月額制の比較）
