- 構造: 英字タグライン + 日本語H1 + サブコピー + WEB予約CTA

```
[タグライン] Devoted to Natural Beauty
[H1]         あなたの自然な美しさを、引き出す場所
[サブ]       ○○市にある完全個室のサロン。丁寧なカウンセリングで、
             あなただけのケアを。
[FVバッジ]   開業○年 / 累計○○名 / リピート率○%
```

## 使用したCTA

| CTA-ID | 配置位置 | 文言 |
|--------|---------|------|
| CTA-008 | ヘッダー固定ナビ | 「ご予約」（カラー強調ボタン） |
| CTA-009 | FV直下・最終CTA | [WEB予約する] [LINEで相談] の2択 |
| CTA-006 | スマホ固定フッター | WEB予約+LINE 2カラム固定 |
| CTA-001 | 施術メニュー後 | 「LINEでこのメニューを予約する」 |

**CTA総数: 5個（FV / ヘッダー / 中盤 / 最終 / スマホ固定）**

## 使用したPattern

- **IND-001**: 美容LP全般パターン
  - ブランディング重視（英語タグライン + 哲学コピー）
  - 差別化施術方式の明示
  - LINE予約がメインCVアクション
  - 初回体験価格の必須明示
  - スタッフ顔出し+資格の信頼性設計

- **Design System**:
  - カラー: `--lux-bg: #faf7f3` / `--lux-gold: #c8a56a`（ラグジュアリーパレット）
  - フォント: Noto Serif JP（見出し）+ Cormorant Garamond（英字）
  - カード: `.card`（ホバーリフト）

## LP構成

```
Section 1: FV
  ↳ FV-009（英語哲学型）
  ↳ サロン内観 or 施術シーン写真フルスクリーン
  ↳ CTA-008（ヘッダー）+ CTA-009（FV直下）

Section 2: 実績・信頼数字 [bg-ivory]
  ↳ stats-grid 4点
     リピート率○% / 累計○名 / 営業○年 / 満足度4.9

Section 3: 施術メニュー [bg-white]
  ↳ grid-3（3メニュー）
  ↳ 写真 + メニュー名 + 初回体験価格 + 通常価格
  ↳ CTA-001（LINEで予約）

Section 4: こだわり・差別化 [bg-white]
  ↳ 左右交互レイアウト（素材/技術/空間）

  ↳ 顔写真 + 名前 + 資格 + 一言コメント
  ↳ IND-001の「スタッフ顔出し必須」原則

Section 6: お客様の声 [bg-ivory]
  ↳ review-card × 3（イニシャル+年代+口コミ）
  ↳ ★★★★★ + blockquote

Section 7: FAQ [bg-white]
  ↳ アコーディオン型（Design System > FAQコンポーネント）
  ↳ 「初めてでも?」「強引な勧誘?」「痛みは?」等5点

Section 8: アクセス [bg-ivory]
  ↳ grid-2（Googleマップ + 店舗情報テーブル）

Section 9: 最終CTA [bg-dark]
  ↳ CTA-009（WEB予約+LINEの2択全幅）
  ↳ CTA-006（スマホ固定フッター）
```

## 採用理由

- FV-009選定: 高単価サロンのブランド感を英語哲学型で演出。「ここにしかない世界観」を1枚で伝える
- CTA-009選定: IND-001で確認済みの「WEB予約+LINE 2択」が美容系標準パターン。Locco D/56でCTA16/20の高スコアを確認
- CTA-008選定: ヘッダー固定ナビでの常時予約導線確保。FAIL-001（ヘッダー予約ボタンなし）の回避

## 想定CVポイント

1. **FV直下CTA（CTA-009）**: 最初のWEB予約ボタンタップ
2. **施術メニューCTA（CTA-001）**: 「このメニューで予約したい」というタイミング
3. **最終CTA（CTA-009）**: LP最下部での最後の意思決定
4. **スマホ固定フッター（CTA-006）**: スクロール中の常時LINE誘導

**主要CV動線**: LINE予約（コミュニケーション重視層） / WEB予約（即決層）

- 構造: 現役スタッフ複数人の顔写真スライド + 職種タグ + 低ハードルCTA

```
[スライド]   スタッフ顔写真カルーセル（3〜5名）

[安心文言]   しつこい連絡なし / 採用担当が直接対応
```

## 使用したCTA

| CTA-ID | 配置位置 | 文言 |
|--------|---------|------|
| CTA-014 | FV / 最終CTA | 「まずは話だけ聞いてみる（見学OK）」 |
| CTA-001 | スタッフ紹介後 | 「LINEで気軽に質問する」 |
| CTA-006 | スマホ固定フッター | [話を聞く] [LINE相談] の2択 |

**CTA総数: 4個**

## 使用したPattern

- **IND-003**: 採用LP全般パターン
  - 低ハードルCTA必須（「応募」より「話を聞く」）

  - スタッフ顔写真+本音コメント
  - 選考フロー可視化（4ステップ）

- **IND-004**: 福祉・住み込み採用パターン
  - 有給取得率・産育休実績の数字明示
  - 「しつこい連絡なし」安心文言の複数配置

- **Design System**:
  - カラー: `--hr-accent: #f97316`（オレンジ / 採用系）
  - フォント: Noto Sans JP（フレンドリー系）
  - .flow-steps（選考フロー可視化）

## LP構成

```
Section 1: FV
  ↳ FV-013（スタッフ顔写真カルーセル）
  ↳ 未経験OK / ブランクOK のターゲットバッジ
  ↳ CTA-014（まずは話だけ）

Section 2: 職場の魅力3点 [bg-white]
  ↳ card--feature × 3
  ↳ 研修制度 / 有給取得率○% / 産育休取得率○%

Section 3: 1日のスケジュール [bg-ivory]
  ↳ timeline形式（daily-schedule）
  ↳ 「介護スタッフの1日」可視化

Section 4: スタッフの声 [bg-gray]
  ↳ staff-voice-card × 3
  ↳ 顔写真+名前+年齢+「入社前の不安→実際は」

Section 5: 給与・待遇 [bg-white]
  ↳ conditions-grid（4区画: 給与/勤務時間/休日/福利厚生）
  ↳ モデル月収例（数字明示）
  ↳ IND-003「給与レンジ必須」準拠

Section 6: 応募フロー [bg-ivory]
  ↳ flow-steps（4ステップ）
  ↳ 相談→見学→面談→入社
  ↳ 「相談だけでもOK」のアピール

Section 7: FAQ [bg-white]
  ↳ アコーディオン × 5点
  ↳ 「未経験OK?」「しつこい連絡?」「いつから?」

Section 8: 最終CTA [bg-recruit]
  ↳ CTA-014（メイン: まずは話だけ）
  ↳ CTA-001（サブ: LINEで相談）
  ↳ 安心文言セット（履歴書不要・連絡しつこくない・直接対応）
```

## 採用理由

- FV-013選定: 採用LPの最大不安は「職場の人間関係」→ スタッフ顔写真複数で「会う人」を最初に見せることで不安を最大限除去
- CTA-014選定: 介護・福祉職は転職に慎重な層が多い。「応募」より「話だけ聞く」が唯一の低ハードルCTA

## 想定CVポイント

1. **FV直下CTA（CTA-014）**: 「まず話を聞く」という最低ハードルCV
2. **スタッフ声セクション後のCTA**: 「この人たちと働きたい」という感情的なタイミング
3. **スマホ固定フッター（CTA-006）**: スクロール中のLINE誘導

# 【CASE-03】医療・整骨院LP

## 使用したTemplate

`knowledge/templates-production/medical-template.md`

## 使用したFV

**FV-014**: 症状共感型FV
- 用途: 整体院・接骨院・鍼灸など「症状→改善」文脈の医療LP
- 構造: 症状バッジ群 + 共感コピー + 初回予約CTA

```
[症状バッジ] 肩こり / 腰痛 / 頭痛 / 自律神経
[H1]         その不調、もう我慢しなくていい
[サブ]       ○○市○○院は、根本原因から改善を目指す整体院です。
             「どこに行っても良くならない」方こそ、一度ご相談ください。
[CTA]        初回予約はこちら（当日OK）
[安心]       初回カウンセリング無料 / 予約優先制
```

## 使用したCTA

| CTA-ID | 配置位置 | 文言 |
|--------|---------|------|
| CTA-012 | 中盤・最終CTA | Web予約+LINE相談+電話の3チャンネル |
| CTA-006 | スマホ固定フッター | [Web予約（24時間）] [電話] の2択 |

**CTA総数: 4個（FV単独 + 中盤3CH + 最終3CH + スマホ固定）**

## 使用したPattern

- **IND-005**: 医療・整体系パターン
  - 薬機法準拠の表現ルール（断定禁止）
  - 症状共感→解決提示の流れ必須
  - 院長・スタッフ顔写真+資格の信頼性設計
  - 料金の透明性（初回体験価格必須）

- **Design System**:
  - カラー: `--med-accent: #2eb88a`（グリーン / 安心・回復）
  - review-card（口コミ / 薬機法対応表現）
  - 3チャンネルCTAコンポーネント（CTA-012）

## LP構成

```
Section 1: FV
  ↳ FV-014（症状共感型）
  ↳ 症状バッジ（肩こり/腰痛/頭痛/自律神経）
  ↳ 初回予約単独CTA（当日OK訴求）

Section 2: 症状チェックリスト [bg-medical-lt]
  ↳ 「こんな症状でお困りではないですか？」
  ↳ check-list × 6〜8項目
  ↳ 「そのお悩み、当院で解決できるかもしれません」bridge文

Section 3: 当院の特徴3点 [bg-white]
  ↳ card--feature × 3
  ↳ 根本原因アプローチ / ○○資格保有 / 通いやすい料金

Section 4: 施術内容・料金 [bg-gray]
  ↳ treatment-list（施術名+説明+時間+価格）
  ↳ 初回限定価格を必ず明示（IND-005原則）

Section 5: 院長・スタッフ紹介 [container--sm]
  ↳ doctor-card（顔写真+役職+名前+資格+メッセージ）

Section 6: お客様の声 [bg-ivory]
  ↳ review-card × 3
  ↳ 薬機法対応: 「症状が楽になりました」表現
  ↳ ※個人の感想 表記必須

Section 7: アクセス [bg-white]
  ↳ grid-2（Googleマップ + 診療時間テーブル）
  ↳ 駐車場情報必須（IND-005原則）

Section 8: FAQ [bg-gray]
  ↳ 医療系必須5点
  ↳ 「保険は?」「何回通う?」「当日OK?」

Section 9: 最終CTA
  ↳ CTA-012（3チャンネル: Web予約+LINE+電話）
  ↳ 「初回カウンセリング無料 / 当日予約対応可」
```

## 採用理由

- FV-014選定: 整体・接骨院は「症状を抱えた人が検索する」業態。症状バッジ + 共感コピーでFV到達時点で「これは自分向け」と感じさせる
- CTA-012選定: 医療系は年代・習慣によって予約方法が異なる。Web予約（若年層）+LINE（質問したい層）+電話（シニア・詳細確認層）の3チャンネルで全年代を取りこぼさない

## 想定CVポイント

1. **FV Web予約CTA**: 「今すぐ予約したい」決断が早い層
2. **施術・料金セクション後のCTA-012**: 「料金を確認して決断」タイミング
3. **最終3チャンネルCTA（CTA-012）**: 年代・好みに応じた3択

```

Section 2: こだわり・コンセプト [bg-dark-restaurant]
  ↳ grid-2（産地写真 + こだわりテキスト）
```

## 想定CVポイント

1. **宴会プランCTA（CTA-016）**: 幹事が「これで決まり」と予約ボタンを押すタイミング
2. **電話確認（CTA-010）**: 「席数・日程を確認してから予約したい」電話タップ
3. **スマホ固定フッター（CTA-006）**: スクロール中の即時来店・予約

## 発見されたギャップ

### GAP-001 ⚠️ スタッフ顔写真カルーセル（FV-013）の実装仕様未定義

**業種**: 採用  
**影響**: FV-013はコンセプトのみ記録。カルーセルのCSS/JSは未実装

**不足内容**:
```
- 自動スライド vs 手動スライド の設計指針なし
- スライドインジケーター（ドット）の実装仕様なし
- スマホでのスワイプ対応の記述なし
- 各スライドの情報量（名前/職種/一言のレイアウト）の仕様なし
```

**解決策**: Design System に `.hero-carousel` コンポーネントを追加（CSS Scroll Snap + JS推奨）

---

### GAP-002 ⚠️ 問い合わせ・予約フォームの実装仕様なし

**業種**: 採用・医療・美容（共通）  
**影響**: LPからの問い合わせフォームがどのサービスを使うか、またはどう実装するかが未定義

**不足内容**:
```
- フォームサービス選定指針なし（Google Form / Formrun / Typeform等）
- HTML自作フォームのスタイル定義不足（Design Systemには入力フィールドのbaseしかない）
- バリデーション（必須項目・メール形式チェック）の実装仕様なし
- サンクスページ設計（送信後のUX）の指針なし
```

**解決策**: CTA-015（インライン資料DLフォーム）を美容・採用・医療にも拡張する汎用フォームコンポーネントを Design System に追加

---

### GAP-003 ⚠️ 医療・整骨院のビジュアル素材ガイドラインなし

**業種**: 医療  
**影響**: 院内写真・スタッフ写真の撮影・入手方法の指針がない

### GAP-005 ❌ アニメーション仕様がfade-inのみ

**業種**: 全業種

**不足内容**:
```
- カウントアップアニメーション（数字実績の「0から増えるアニメ」）なし

- スティッキーヘッダーのスクロール時背景変化（透明→白）の仕様なし
- ローディングスケルトンUI なし
```

**解決策**: Design System の アニメーションセクションに `.count-up` / `.sticky-header-transition` を追加
