## 最重要ルール

```
1. Claude Codeは画像生成を行わない
2. 画像生成が必要な場合は ChatGPT（DALL·E 3）のみを使用する
3. Claude Codeの役割: 画像設計 / 素材選定 / 検索ワード作成 / 生成プロンプト作成 / 配置指示
```

## 素材調達フロー（LP制作時）

```
STEP 1: 業種確定
    ↓
STEP 2: industry-asset-patterns.md で業種の「必要素材一覧」確認
    ↓
STEP 3: asset-checklist.md で素材チェックリスト作成
    ↓
STEP 4: 各素材の調達方法を決定
    │
    ├─ CLIENT提供が可能 → クライアントへ素材リストを共有
    ├─ フリー素材で対応 → photo/illustration-source-library.md で検索ワード確認
    └─ ChatGPT生成が必要 → chatgpt-image-prompt-library.md でプロンプト確認
    ↓
STEP 5: 素材が揃ったらasset-checklist.md Phase 2チェック実施
    ↓
STEP 6: HTMLに組み込み → asset-checklist.md Phase 3チェック実施
```

## 画像ファイル管理規則

### ファイル形式

| 用途 | 推奨形式 | 理由 |
|------|---------|------|
| 写真（本番） | WebP（JPEGフォールバック）| ファイルサイズ削減・高品質 |
| 写真（開発） | JPEG | 確認用として |
| イラスト・アイコン | SVG | スケーラブル・軽量 |
| 透過画像 | PNG（またはWebP）| 透過対応 |
| OGP | JPEG または PNG | SNS互換性 |
| ファビコン | ICO / PNG / SVG | 複数形式対応推奨 |

### ファイルサイズ目安

| 配置場所 | 最大サイズ目安 | 推奨解像度 |
|---------|-------------|----------|
| FV画像 | 300KB以下 | 1920×1080px |
| セクション画像 | 150KB以下 | 1200×800px |
| スタッフ写真 | 100KB以下 | 600×600px |
| OGP画像 | 200KB以下 | 1200×630px |
| アイコン（SVG）| 10KB以下 | — |

### フォルダ構成

```
[案件名]-lp/
├── index.html
└── images/
    ├── fv-main.jpg（またはwebp）
    ├── fv-main.webp
    ├── service-01.jpg
    ├── service-02.jpg
    ├── staff-01.jpg
    ├── ogp.jpg
    └── [その他]
```

## ChatGPT生成画像 使用ガイドライン

### 使用可能なケース

- フリー素材やクライアント提供で代替できない場合
- 特定の業種・シチュエーションの人物写真
- プレースホルダー（クライアント素材待ちの間）

### 使用禁止ケース

- 実在の人物・施設として誤解を招く表示
- 医療行為の「効果保証」的な文脈での使用
- 景品表示法・薬機法に抵触する可能性がある文脈

### 生成後チェックリスト

```
□ 手・指の本数が自然か（5本であることを確認）
□ 顔の造形が崩れていないか
□ 背景に不自然な歪みがないか
□ 画像内に意図しないテキスト・ロゴがないか
□ 業種的にNGな表現がないか
□ 人物の目線・表情が自然か
□ 医療系: 治療効果を示唆する表現がないか
□ 実在の店舗・施設に見せかけていないか
```

## フリー素材 ライセンス確認ルール

使用前に必ず確認すること。

```
確認項目:
□ 商用利用が許可されているか
□ 帰属表記（クレジット）が必要か → 必要な場合はフッターに記載
□ 加工・編集が許可されているか
□ LP・Webサイトへの使用が明示的に許可されているか
□ 利用規約の変更がないか（特に頻繁に利用するサイト）
```

| サイト | 帰属表記 | 商用 | 加工 |
|--------|---------|------|------|
| Unsplash | 不要（任意推奨）| ○ | ○ |
| Pexels | 不要（任意推奨）| ○ | ○ |
| Pixabay | 不要 | ○ | ○ |
| unDraw | 不要 | ○ | ○ |
| Storyset（無料）| 必要 | ○ | ○ |
| ManyPixels | 不要 | ○ | ○ |
| Lucide | 不要（ISC）| ○ | ○ |
| Heroicons | 不要（MIT）| ○ | ○ |

## アンチパターン（禁止事項）

```
✗ Claude Codeで直接画像を生成しようとする
✗ 素材確認なしにコーディングを開始する
✗ ライセンス未確認のフリー素材を使用する
✗ クライアント許諾なしの人物写真を使用する
✗ ロゴ・商標が写り込んだ画像をそのまま使用する
✗ 薬機法・景表法に抵触するBeforeAfter画像を無確認で使用する
✗ 解像度不足（800px以下）の画像をFVに使用する
✗ 写真とイラストを同一セクションに混在させる
✗ ChatGPT生成画像を「実在の人物・施設」として提示する
```

## Learning Reason

### 保存理由
素材管理は全LP制作で必ず発生するが、場当たり的な判断が品質のばらつきを生んでいた。
Asset Management Systemとして体系化することで、制作品質の均一化を実現する。

### 再利用価値
全業種・全制作物（LP/HP/WP）で適用可能。

### 特に優れている点
- Claude Codeと人間とChatGPTの役割分担を明確化
- 業種別の必要素材を事前に洗い出すことで制作遅延を防ぐ
- ライセンス・薬機法チェックを標準フローに組み込む
