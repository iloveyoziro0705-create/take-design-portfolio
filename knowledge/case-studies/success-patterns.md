## PATTERN-001: 固定フッターCTA（全LP共通）

**採用率**: 6/6案件（100%）

### 内容

スクロール一定距離後（500px前後）に表示される固定ボタンCTAを全LPに実装。

```css
.fixed-cta {
  position: fixed;
  bottom: 0;
  left: 0;
  right: 0;
  z-index: 100;
  transform: translateY(100%);
  transition: .3s;
}
.fixed-cta.show {
  transform: translateY(0);
}
```

```js
window.addEventListener('scroll', () => {
  fixedCta.classList.toggle('show', window.scrollY > 500);
});
```

### なぜ採用するか

LPをスクロールする途中でCTAから離れてしまう「CTA見失い問題」を根本解消。
スクロール500px以降は常にCTAが視野内にある状態を作ることで、CVの取りこぼしを防ぐ。

### 業種別バリエーション

| 業種 | ボタンテキスト | カラー |
|------|------------|-------|
| 美容サロン | LINE無料相談 / WEB予約 2択 | ゴールド |
| キャリア相談 | 無料相談を申し込む | ブルー |
| AIスクール | 無料相談 | ネイビー |
| 保育士プラットフォーム | 無料で口コミを見る | ピンク |
| 住み込み求人 | 求人を探す | オレンジ |
| 飲食 | WEB予約する | ゴールド |

## PATTERN-002: FadeUp アニメーション（全LP共通）

**採用率**: 5/6案件（83% — restaurantは未実装）

### 内容

IntersectionObserverによるスクロール連動フェードアップアニメーション。

```css
.fadeUp {
  opacity: 0;
  transform: translateY(30px);
  transition: opacity .6s ease, transform .6s ease;
}
.fadeUp.visible {
  opacity: 1;
  transform: translateY(0);
}
```

```js
const observer = new IntersectionObserver(entries => {
  entries.forEach(e => {
    if (e.isIntersecting) e.target.classList.add('visible');
  });
}, { threshold: 0.15 });
document.querySelectorAll('.fadeUp').forEach(el => observer.observe(el));
```

### なぜ採用するか

セクションが突然現れるより「滑らかに登場する」体験がブランド品質を高める。
実装コストが低い割にUXへの貢献度が高く、全業種で採用すべき標準実装。

## PATTERN-003: ヒーロー信頼バッジ（美容・採用系で高採用率）

**採用率**: 3/6案件（美容・保育士・住み込み）

### 内容

FVまたはHero直下に「数値バッジ」を3〜4点並べる。

**Beauty Salon**: ★4.9 / 施術3,000件 / リピート92%

**ミライ寮**: 10,000人紹介 / 満足度95% / 3,000件 / 最短2日  

```html
<div class="hero-badges">
  <span class="badge">★4.9</span>
  <span class="badge">施術3,000件以上</span>
  <span class="badge">リピート率92%</span>
```

### なぜ採用するか

FV到達時点で「すでに評価されている」ことを伝え、初期信頼を確立する。
「第三者評価の先行提示」が後続セクションの読了率を高める。

## PATTERN-004: 悩み共感セクション（B2C全般で高採用率）

**採用率**: 5/6案件（restaurant以外すべて）

### 内容

「こんな悩みありませんか？」または「こんな不安はありませんか？」のチェックリスト形式。

**採用例**:
- 美容サロン: Worry セクション（肌悩みチェック）
- AIスクール: 悩み訴求（AI不安チェック）
- Career Compass: Recommend（モヤモヤチェックリスト6点）
- 保育士のミカタ: 職場不安の代替として口コミ訴求
- ミライ寮: Problems（6カード: お金/住まい/速度/環境/スキル/地方）

### なぜ採用するか

ユーザーが「これは自分のことだ」と自己認識する「自分事化」の最も効率的な手法。
チェックリスト形式は「当てはまる」という能動的確認を促し、次セクションへの興味を引き出す。

---

## PATTERN-005: Before / After 比較設計

**採用率**: 3/6案件（Career Compass / AIスクール / ミライ寮）

### 内容

「サービス利用前の状態」と「利用後の状態」を視覚的に対比する。

**Career Compass**: 2カラム（Before / After テキスト比較）  
**AIスクール**: 受講生変化セクション  
**ミライ寮**: タイプ別体験談にミニBefore/After付き  

### なぜ採用するか

「自分がどう変われるか」という期待値を具体化することで、CVへの意思決定を加速する。
特に「現在困っている→解決後の未来」という変化を訴求するサービスで必須。

## PATTERN-006: 2択CTA設計（美容・求人系で高採用率）

**採用率**: 3/6案件（美容 / 保育士 / ミライ寮）

### 内容

2つのCTAを並べ、ユーザーに「どちらかを選ぶ」体験をさせる。

**美容サロン**: LINE無料カウンセリング ／ WEB予約  
**保育士のミカタ**: 無料で口コミを見る ／ 人気ランキングを見る  
**ミライ寮**: 無料診断スタート ／ 求人一覧を見る  

### なぜ採用するか

「何もしない」よりも「どちらにするか選ぶ」方がCV率が高い（決断コストの変換）。
LINEとWEBなど「接触方法が異なる2ターゲット」を同時に包括できる。

## PATTERN-007: 数値Stats（プラットフォーム・スクール系）

**採用率**: 4/6案件（保育士 / Career Compass / ミライ寮 / AIスクール）

### 内容

「〇〇件以上」「〇〇%」「〇〇人」など実績数値を強調表示するStats セクション。

**Career Compass**: 無料 / 60分 / 初心者OK / 勧誘なし（条件の数値化）  
**ミライ寮**: 10,000人 / 95% / 3,000件 / 最短2日  
**保育士のミカタ**: 口コミ100万件以上 / 無料 / 比較 / ランキング  

### なぜ採用するか

漠然とした「多くの方が利用」より「10,000人が相談済み」の具体数値の方が信頼性が高い。
FV通過後の最初の「疑問（本当に信頼できるか？）」を数値で解消するタイミングに配置。

## PATTERN-008: FAQ セクション（全業種必須）

**採用率**: 6/6案件（100%）

### 内容

「よくある質問」セクション（3〜5問）。形式はアコーディオン or 静的テキスト。

| 業種 | 主なFAQ内容 |
|------|-----------|
| 美容サロン | 初回の流れ / 男性OK? / キャンセルポリシー |
| キャリア相談 | 無料?/ 未経験?/ 購入必要?/ 顔出し? |
| AIスクール | どんな人向け?/ 料金?/ 期間?/ Mac/Win対応 |
| 保育士のミカタ | 無料?/ 個人情報?/ 保育園側への影響? |
| 住み込み求人 | 予約必要?/ 個室?/ 記念日対応?（restaurant） |
| 飲食 | 予約必要?/ 個室?/ 記念日 |

### なぜ採用するか

CVの直前に「最後の不安」を除去する最終防衛ライン。
FAQがないと「気になったけど調べるのが面倒」で離脱するユーザーを拾えない。

## PATTERN-009: 中間CTA（長尺LP限定）

**採用率**: 2/6案件（Career Compass / ミライ寮）

### 内容

LPの中間（スクロール50〜70%位置）にインラインCTAブロックを1〜2箇所配置。

**Career Compass**: 「今のモヤモヤを言葉にしてみませんか？」× 2箇所  
**ミライ寮**: タイプ別分岐LP内に各タイプCTAを配置  

### なぜ採用するか

長尺LP（1,000行以上）では最終セクションまでスクロールしないユーザーが多い。
中間でのCTA配置により「この時点で申し込みたいユーザー」の取りこぼしを防ぐ。

## まとめ: 若井式LP設計の7原則

1. **固定フッターCTA は全LP必須** — スクロール500px後に表示（PATTERN-001）
2. **FadeUpアニメーションで品質を表現** — IntersectionObserver実装（PATTERN-002）
3. **FVバッジ3点で初期信頼を確立** — ★評価×実績件数×特徴数値（PATTERN-003）
4. **悩み共感で自分事化させてから解決を提示** — 「こんな悩みありませんか？」先行（PATTERN-004）
5. **FAQ は離脱防止の最終防衛** — 業種特有の不安を5問で解消（PATTERN-008）
6. **長尺LP（1,000行以上）には中間CTAを必ず設置** — スクロール途中での取りこぼし防止（PATTERN-009）
7. **カラートークンは業種標準で固定** — 毎回ゼロから決めず標準トークンを適用（PATTERN-011）
