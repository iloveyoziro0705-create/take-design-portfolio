# 若井式LP設計パターン — 改善余地・失敗パターン集

**抽出元**: 自社制作6案件の改善余地・課題の共通パターン分析  
**更新日**: 2026-06-24  
**対象案件**: 保育士のミカタ / Career Compass / AI MASTER SCHOOL / Lumière Beauty Salon / Restaurant Opening LP / ミライ寮

---

## IMP-001: スタッフ・プロフィール写真がSVGイラスト or 未掲載

**発生案件**: 美容サロン（未掲載）/ Career Compass（SVGイラスト）/ AIスクール（顔写真なし）  
**影響度**: 高

### 何が起きたか

制作時点で実写スタッフ写真が入手できず、SVGイラストプレースホルダーまたはスタッフ写真なしで納品した。

### なぜ問題か

- 個人サービス・美容・コーチング系では「この人に頼んで大丈夫か」という不信感が最大障壁
- 実際の顔写真は「この人がいる」という実在感・親近感を生む
- SVGイラストは「架空のサービス感」を与え、CV率を下げる可能性がある

### 今後の対処

- 制作前に「プロフィール写真の実写素材の有無」を必ず確認する
- 実写がない場合は「写真が届いたら差し替える前提のプレースホルダー」として明示
- スタッフ写真は最初から実写で用意させる（後から差し替えると工数が高い）

## IMP-002: 料金・費用情報が未掲載

**発生案件**: AIスクール（料金未掲載）/ Career Compass（相談料は無料だが次のステップ費用が不明）  
**影響度**: 高

### 何が起きたか

「無料相談」がCTAなのに、その後の費用が一切明示されていない。

### なぜ問題か

- スクール系では「高額商品を売られないか」という不安が問い合わせ前の最大ハードル
- 料金が不明だと「申し込んだら高額提案をされる」という警戒感でCV率が下がる
- 問い合わせの質も下がる（費用感が合わない人が来て、現場負担が増える）

### 今後の対処

- 料金または「〇〇円〜」の目安を必ず掲載する
- 完全無料サービスは「無料の理由」を明示する（「完全無料・勧誘なし」のバッジ）
- 高額サービスは「個別見積もり」でも「おおよその相場感」を文章で補足

## IMP-003: ヘッダーCTA・固定ヘッダーの欠如

**発生案件**: Restaurant Opening LP（ヘッダーなし）  
**影響度**: 高

### 何が起きたか

Restaurant LPでは固定ヘッダー自体を省略した設計になっており、スクロール中にナビ・CTAが消える。

### なぜ問題か

- ユーザーが「このお店に行こう」と思った瞬間にCTAが視野にない
- スクロール中の「予約したいときに予約できない」という機会損失

### 今後の対処

- 全LPに固定ヘッダー + ヘッダー内CTAを標準実装する
- シンプル志向のLPでも最低限「ヘッダー右端にCTAボタン」は設ける
- 固定フッターCTA（PATTERN-001）と組み合わせて二重カバー

## IMP-004: FAQの数・内容が不十分

**発生案件**: Restaurant（3問のみ）/ AIスクール（FAQ未実装）  
**影響度**: 中

### 何が起きたか

FAQが3問以下、または未実装のLPが存在した。

### なぜ問題か

- FAQはCVを阻む「最後の疑問」を除去する唯一のセクション
- 3問では業種特有の不安（料金・期間・対象者適合性）を解消しきれない
- FAQがないと「気になったが調べるのが面倒」で離脱するユーザーが続出

### 今後の対処

- FAQ は最低5問、理想は8〜10問を設ける
- 業種テンプレートを用意し、毎回ゼロから考えない
- アコーディオン実装で視覚的な圧迫感なく多問掲載

**業種別FAQ必須問テンプレート**:

| 業種 | 必須FAQ |
|------|--------|
| 美容サロン | 初回の流れ / 男性OK? / キャンセル / 料金 / 痛みはあるか |
| スクール | 向いている人は? / 受講期間 / 料金 / 未経験OK? / Mac/Win |
| 採用・求人 | 未経験OK? / 経歴 / 応募方法 / 選考期間 |
| 個人相談 | 無料?/ 何回?/ 強引な勧誘?/ オンライン?/ 顔出し必要? |
| 飲食 | 予約必要? / 個室? / アレルギー / 記念日 / キャンセル |

## IMP-005: 口コミ・体験談が仮データ・匿名・少数

**発生案件**: 全案件（程度の差はあれ共通課題）  
**影響度**: 高

### 何が起きたか

- 口コミが「A様」「K様」などの匿名のみ（Career Compass）
- 口コミが3件のみ（Restaurant、美容サロン）
- 口コミが仮データ（保育士のミカタ — サービス立ち上げ段階の制約）

### なぜ問題か

- 「〇〇様」の匿名口コミより「20代/IT職/副業検討」の属性付きの方が信頼性が高い
- 3件では「一部の人だけの意見」に見える。最低10件以上で「多くの声がある」と認識される
- 仮データは「このサービスに実績があるのか？」という疑念を生む

### 今後の対処

- 口コミは最低10件以上を確保してからLP制作に入る
- 匿名の場合でも「属性情報（年代/職業/相談種別）」を付記する
- 写真付き口コミは信頼性が約2倍（写真の実在感が証言を強化）
- Googleレビュー埋め込みで「第三者プラットフォームの口コミ」として信頼性をさらに高める

## IMP-006: Before / After ビジュアルの欠如

**発生案件**: 美容サロン / Restaurant（サービス特性上困難）  
**影響度**: 中

### 何が起きたか

美容サロンLPに施術のBefore/After写真がなく、効果の可視化ができていない。

### なぜ問題か

- 美容・ダイエット系は「どれだけ変われるか」が最大の購買動機
- Before/Afterがないと「本当に効果があるのか不明」という疑念が残る

### 今後の対処

- 美容・医療・ダイエット系では Before/After 写真の確保を制作前に必ず確認
- 薬機法対応が必要な場合は「個人差があります」の明記とともに掲載
- 写真が入手困難な場合は「施術の流れ」セクションで代替（IND-003参照）

## IMP-007: 施術・サービスの流れが未掲載

**発生案件**: 美容サロン / AIスクール（部分的）/ Career Compass（Flow あり → 良例）  
**影響度**: 中

### 何が起きたか

「初回でどんな流れになるのか」が不明なLPが複数存在した。

### なぜ問題か

- 「初めての人が来店 or 申し込んだら何が起きるか」が不明だと「やってみたいけど怖い」で離脱
- 特に美容・医療・スクールなど「身体・時間・お金を預ける」サービスではフロー開示が必須

### 今後の対処

- 「初回の流れ（3〜5STEP）」セクションを全サービス系LPに標準装備する
- Career Compass の「Flow（4STEP）」が最良の参照例
- 「申込 → 日程調整 → 相談 → フォローアップ」のような具体的なSTEPで安心感を作る

## IMP-008: スマートフォン専用CTAテキストの未最適化

**発生案件**: 保育士のミカタ（PC/SP共通テキスト使用）  
**影響度**: 低〜中

### 何が起きたか

PC向けに設計された長いCTAテキストをモバイルでもそのまま使用。

### なぜ問題か

- モバイルは画面幅が狭く、長いCTAテキストが折り返したり小さくなる
- 「無料で口コミを確認してランキングも見てみる」→ モバイルでは「今すぐ見る」に短縮すべき
- タップ操作に最適化されたCTAは「大きく・短く・行動語」が原則

### 今後の対処

```html
<span class="pc-only">無料で口コミを見る（登録3分）</span>
<span class="sp-only">今すぐ無料で見る</span>
```

- モバイルとPC でCTAテキストを使い分ける
- スマホCTAは12文字以内を目安

## IMP-009: 予約・申込フローの摩擦

**発生案件**: Career Compass（メール日程調整）/ Restaurant（外部遷移）/ 美容サロン（LINE遷移）  
**影響度**: 高

### 何が起きたか

「CTAを押した後に予約完結しない」フローが複数案件で残存。

| 案件 | 問題 |
|-----|------|
| Career Compass | フォーム送信後、「メールで日程調整」というステップが残る |
| Restaurant LP | WEB予約リンクが外部ページへ遷移（LP離脱が発生） |
| Beauty Salon | LINE遷移後にスタッフ対応待ちが発生（LINE既読まで完結しない） |

### なぜ問題か

CVの「モーメント」（申し込みたいという気持ちのピーク）を逃さないことが最重要。
「次のステップが残る」と感じた瞬間に離脱率が上がる。

### 今後の対処

- **最優先**: Calendly / STORES予約など「LP内カレンダー予約完結」を実装（CTA-017参照）
- Restaurant系: inlineフォーム（名前/日時/人数/備考）をCTAセクション内に実装
- LINE CTAは「LINEアカウントに友達追加するだけ」という最小アクション設計

## IMP-010: LPの重量・パフォーマンス最適化

**発生案件**: ミライ寮（2414行）/ 美容サロン（933行）  
**影響度**: 中

### 何が起きたか

制作を重ねるほどLPが大規模化し、パフォーマンスへの考慮が後手になった。

### なぜ問題か

- スクロール量が多いほど離脱率が上がる（特にモバイル）
- 外部フォントの読み込み数が多いとFCP（First Contentful Paint）が遅くなる
- 大量のanimationやobserverが積み重なるとスクロールのジャンク（引っかかり）が発生

### 今後の対処

- 1,500行を超えたら「セクション削減 or ページ分割」を検討する
- Google Fontsは `display=swap` + preconnect で高速化
- IntersectionObserverは `once: true` オプション（観察済み後は停止）で負荷軽減
- Unsplash等の外部画像は `loading="lazy"` を付与

## IMP-011: 実写素材ゼロでのLP公開

**発生案件**: Restaurant（全画像がUnsplash仮素材）/ AIスクール（スタッフ写真なし）  
**影響度**: 高（実案件では致命的）

### 何が起きたか

ポートフォリオ制作のため、Unsplash素材または仮イラストのままLP公開。

### なぜ問題か

実案件で同様の状態で公開すると:
- 「この写真は自社のものではない」とユーザーが気づいた場合、信頼を失う
- Unsplash商用利用は規約上問題ないが、「他の多くのサイトと同じ写真」がバレると安っぽく見える
- 写真の印象とサービスの実態が乖離するリスク

### 今後の対処

- 実案件では「必ず実写素材を先に確保してからLP制作に入る」を原則化
- 素材が届く前に制作する場合は「仮素材 → 実写に差し替え」を工程に明記
- CLAUDE.mdの「イラスト・画像の使い方」フローを遵守する
