# design-spec.md — 建設会社LP デザイン解析仕様書

元画像（○○建設株式会社／地域密着・自社施工LP、852×1847px相当）を構造的に解析し、Figmaで100%編集可能なデザインシステムとして再構築するための仕様書。**画像を模写するのではなく、画像の背後にあるデザインルールを抽出したもの。**

---

## 1. デザインコンセプト

「地域の暮らしと街の安全を、自社一貫施工の職人技術で足元から支える」という**土木・外構・造成の専門工事店**としての信頼を伝えるLP。派手な演出よりも「誠実さ」「実直さ」「現場感」を前面に出し、価格の安さや割引訴求ではなく **無料相談・相見積もり歓迎・強引な営業なし** という"安心して問い合わせられる"心理的ハードルの低さを最優先に設計されている。

## 2. ブランドイメージ

- Navy（濃紺）＝信頼・実直・安全性・法人格の安心感
- Orange（オレンジ）＝行動喚起・温かみ・現場の活気（重機・作業服のイメージ想起）
- 装飾要素は最小限。写真（実際の施工現場・職人）を主役に据え、イラストや過度なグラデーションを避けることで「誇張しない工務店らしい誠実さ」を表現。

## 3. ターゲット

- 一次ターゲット：戸建てを持つ個人（外構・ブロック塀・駐車場舗装・解体等の困りごとを抱える施主）
- 二次ターゲット：造成・擁壁・公共インフラ工事を発注する法人・自治体担当者
→ ヒーローコピーで個人向け（暮らし）と法人向け（街）の両方に触れることで、両ターゲットを同時に受容している。

## 4. 情報設計（ページ全体構造）

```
1. ヘッダー（ロゴ + 常時CTA2種 + メニュー）
2. ヒーロー（対応エリア → コピー → 3つの安心訴求 → Before/After実績）
3. プライマリCTA群（LINE見積り / 電話 / 信頼バッジ3点）
4. 対応工事（6サービスのカードグリッド）
5. 実績スタッツ帯（件数・創業年数・対応スピード）
6. フッター固定CTA（電話 / 見積り）
```

**PASONA的な逆算構造**ではなく、**先に信頼（対応エリア・自社一貫・地域密着）→ 次に行動喚起（CTA）→ 次に詳細（対応工事）→ 最後に裏付け（実績数字）**という「安心 → 行動 → 詳細 → 裏付け」の順で情報を出す、地域密着型工務店LPに典型的な設計。CTAをヒーロー直後・サービス一覧の後の計2回配置し、離脱ポイントごとに行動導線を用意している。

## 5. 視線誘導

- ヘッダー：Z型（ロゴ→CTA→メニュー）
- ヒーロー：F型（コピー左上から下へ）+ 右側写真への視線の流れ（職人の視線・手元の作業がそのままCTAブロックへの視線誘導になる構図）
- サービスカード：写真→アイコン→タイトル→説明の縦Z型を6回反復
- 全体：**大→小→大**のジャンプ率で、ヒーロー見出し（44px）→本文（16px）→CTAボタン（20px）→スタッツ数字（40px）と、要所で情報の重みを切り替えて飽きさせない設計。

## 6. 余白設計・ジャンプ率

- セクション間の縦マージン：`space.16`（64px）を基準に、要素同士は`space.6〜8`（24〜32px）
- ジャンプ率（本文16pxに対する見出し比率）：
  - ヒーロー見出し 44px / 本文16px = **275%**（非常に高いジャンプ率。第一印象のインパクト重視）
  - セクション見出し 32px / 本文16px = 200%
  - スタッツ数字 40px / ラベル14px = 285%（実績の説得力を数字の大きさで担保）
- 余白は「詰めすぎない」が基本方針。カード内padding `space.6`（24px）、カード間gap `space.5〜6`（20〜24px）で、圧迫感のない工務店らしい"誠実な余白"を確保。

## 7. グリッド

| ブレークポイント | 幅 | コンテナ | カラム | マージン | ガター |
|---|---|---|---|---|---|
| Mobile | 〜599px | fluid | 1 | 20px | 16px |
| Tablet | 600〜1023px | 900px | 2 | 32px | 20px |
| Desktop | 1024px〜 | 900px（中央寄せ） | 3 | auto | 24px |

> 元画像は852px幅を基準とした「1カラム中央寄せ型」の工務店LPに典型的な組み方。デスクトップでも横幅を広げすぎず900px程度に収め、左右に無地の背景を敷くことで可読性を保つ設計思想を踏襲した。対応工事セクションのみカードグリッドとして2〜3カラムに展開する。

詳細は[design-token.json](./design-token.json)を参照。要点：

| Token | Hex | 用途 |
|---|---|---|

| Secondary/Accent（Orange600） | `#CC6A1B` | 無料見積りボタン、LINE CTA、強調テキスト、チェックアイコン |
| Gold（Gold500） | `#C9A66B` | 実績スタッツの月桂樹アイコン |
| Background | `#FDFDFC` | ページ全体の背景 |
| Cream | `#F2EFE7` | 信頼バッジ（相見積もり歓迎 等）のピル背景 |
| Border | `#E6E8EC` | 罫線・区切り |

配色は実質**Navy×Orangeの2色構成**で統一され、グレーは文字の階調にのみ使用。彩度の高い色を増やさないことで「工務店の誠実さ」を損なわない。

## 10. 写真比率

- ヒーロー写真：縦長トリミング、比率 約 **3:4〜4:5**（右側55%幅、フレーム上下フルブリード、左端は白背景へグラデーションでフェード）
- Before/After比較：正方形に近い比率 **1:1**、2枚を1つの角丸コンテナ内に隣接配置、中央に再生/切替アイコンを重ねる
- サービスカード写真：横長 **4:3**、カード上部、角丸16px（上2角のみ）

## 11. カード設計

- **サービスカード**：写真(4:3, 上部角丸16px) + 円形アイコンバッジ(写真左下に半分重ねてオーバーラップ配置) + タイトル(Bold 20-24px) + 説明文(2〜3行) を縦積みAuto Layout。カード全体はradius `lg`(16px)、shadow `card`、背景white。
- **Before/Afterカード**：横に2分割、各画像に半透明ラベルタグ（BEFORE=グレー、AFTER=Orange）を左上に重ね、中央に円形の矢印/再生アイコンを配置。
- **円形アイコンバッジ（現地調査無料 等）**：白背景+Navy 1.5px枠線の円、内側にNavyのラインアイコン、下にラベル(2行、強調語のみOrange)。

## 12. ボタン設計

| Variant | 背景 | 文字色 | Radius | Height | 用途 |
|---|---|---|---|---|---|
| Primary（Orange大型） | `orange.600` | 白 | `xl`(20px) | 64px | LINE見積りCTA（最重要） |
| Secondary（Navy大型） | `navy.800` | 白 | `xl`(20px) | 64px | 電話CTA |
| Header-Compact（Navy/Orange 2分割） | 左Navy／右Orange | 白 | `none`（左右いずれかの外側角のみ`md`） | 48px | ヘッダー固定CTA |
| Footer-Split（Navy/Orange 2分割・全幅） | 左Navy／右Orange | 白 | `none` | 56px | 下部固定フッターCTA |
| Outline（メニュー） | 白 | Navy | `none` | 48px | ハンバーガーメニュー |

すべてのボタンにアイコン（左）+ラベル+矢印アイコン(右円形、半透明白)の3点構成を統一。Hover時は`translateY(-2px)`+shadow強調、Active時は`translateY(0)`+shadow減衰、Disabledは彩度を落とし`opacity 0.5`。

## 13. アイコン設計

- スタイル：線幅 **1.75〜2px** のラインアイコン（Feather/Lucide系）で統一
- サイズ：円形バッジ内アイコン 28〜32px／カードアイコン 24px／ボタン内アイコン 20〜24px
- カラー：Navy背景上=白、白背景上=Navy（2値のみ、彩度を持たせない）
- 余白：円形バッジの直径に対しアイコンは約40%、外周に十分な呼吸スペースを確保

## 14. 影（Shadow）

| Token | Offset | Blur | Spread | Color |
|---|---|---|---|---|
| card | 0 / 8px | 24px | -4px | rgba(15,36,64,0.12) |
| cta-button(Orange) | 0 / 10px | 20px | -6px | rgba(204,106,27,0.45) |
| cta-button-navy | 0 / 10px | 20px | -6px | rgba(15,36,64,0.35) |
| icon-badge | 0 / 4px | 12px | 0 | rgba(15,36,64,0.08) |
| sticky-footer | 0 / -4px | 16px | 0 | rgba(0,0,0,0.15) |

影は常に**ブランドカラーを帯びた色付き影**（グレーの汎用影は使わない）。CTAボタンだけ強めのグロー影で視線を集める設計。

## 15. 角丸（Radius）

- sm(8px)：対応エリアピル
- md(12px)：小要素
- lg(16px)：カード、Before/Afterコンテナ
- xl(20px)：大型CTAボタン
- full：円形バッジ、丸アイコン、チェックボックス

## 16. 罫線（Border）

- 円形アイコンバッジ：Navy 1.5px実線
- セクション見出し左右の飾り線：Navy 1px実線、幅可変（テキスト幅に応じてAuto Layoutで伸縮）
- カード：基本罫線なし（shadowのみで浮遊感を出す）

## 17. 背景

- ページ全体：`#FDFDFC`（純白よりわずかに温かいオフホワイト）
- ヒーロー：白 → 右側写真へのグラデーションフェード（`linear-gradient(to right, #FDFDFC 0%, transparent 45%)`を写真に重ねる）
- スタッツ帯：Navy全面ベタ背景（フルブリード、コンテナ幅を超えて画面幅いっぱい）
- フッターCTA：Navy/Orange 2分割ベタ背景、画面幅いっぱい

## 18. 装飾

- セクション見出し（「対応工事」）左右の斜め短線装飾のみ。過度な装飾（波線、ブロブ、抽象図形）は一切なし。
- 実績スタッツの月桂樹（laurel wreath）アイコンが唯一の"権威付け"装飾要素。

## 19. 余白ルール（Spacing System）

`4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48 / 56 / 64 / 80 / 96 / 120 / 160` の15段階スケールに準拠（詳細は design-token.json の `spacing` トークン）。全てのmargin/padding/gapはこのスケール外の値を使わない。

## 20. レスポンシブ思想

- **Mobile-first ではなくコンテンツ幅固定型**：どのブレークポイントでも最大900px程度のセンターカラムを維持し、広い画面では左右に余白（背景色のみ）を持たせる、地域工務店LPに典型的な"読み物として読ませる"思想。
- ヒーローの2カラム構成（コピー+写真）はTablet以上でのみ有効。Mobileでは「コピー→写真」の縦積みに変換し、写真は横幅いっぱい・比率4:5でクロップ。
- 対応工事カードグリッドは Mobile=1列 / Tablet=2列 / Desktop=3列（元画像通り）。
- ヘッダーCTA（電話/見積り）とフッター固定CTAはMobileでは画面下部に`position: sticky`で常時表示し、スクロールへの追従性を最優先にする。
