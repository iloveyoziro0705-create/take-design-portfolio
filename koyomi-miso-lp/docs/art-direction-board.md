## 1. 参考サイト分析（3件・取り入れる要素とその理由）

| 参考サイト | 取り入れる要素 | 本案件への適用 |
|---|---|---|
| 薬膳粥LP（saishunkan） | 白・薄グレー基調の余白設計／素材の「ありのまま」写真／セクション完結ごとの控えめCTA | 背景の「白・生成り」区間の設計思想として採用。CTAは購入訴求ではなくLINE登録に置き換え、同じ「押し付けない」トーンを踏襲 |
| 大地のムクナ（PR TIMES） | 耕作放棄地再生・伝統と科学の融合というブランドストーリー構造／「日常的に使える食」という間口の広さ | ブランドストーリーの「感情のある物語」の型として参照（ただし暦みそは架空案件のため、糸島の実話ではなく「二十四節気に寄り添う味噌蔵の物語」として独自に創作する） |
| 発酵本家の味噌汁LP | 大量画像によるビジュアルストーリーテリング／CTA複数反復（上中下3回） | 「情報量ではなく画像で語る」姿勢は継承するが、**縦長シングルカラムの通販的畳み掛けは採用しない**（ブリーフの「雑誌・読み物」要件と矛盾するため）。CTA複数反復のみ構造として踏襲し、文言を「お得に始める」→「季節の薬膳レシピを受け取る」に置き換え |

**差別化ポイント**: 3サイトはいずれも最終的に「購入」がゴールだが、本案件は**LINE登録という低ハードルCVのブランドメディア型**。そのため通販的な「オファーの畳み掛け」は排除し、雑誌・読み物としての完成度と余白を優先する。

## 2. ブランドコンセプト

### ブランド名
**暦みそ ―KOYOMI―**（確定・変更不可）

### コンセプト
**二十四節気とともにめぐる、食べる暦。**

説明ではなく物語として：

> 味噌は、暮らしの中にそっと季節を運んでくれる存在でした。
> 立春の若菜、夏至の生姜、白露の根菜、冬至のかぼちゃ——
> 二十四節気ごとに変わる食材を、代々受け継がれた発酵の知恵で味噌に仕立てる。
> 「暦みそ」は、忙しい日々の中でも、今日という季節を一杯の味噌汁で感じてもらうためのブランドです。

### Brand Promise（一文）
**「季節のめぐりに寄り添い、毎日の食卓から身体を整える暮らしを届けます。」**

このBrand Promiseを、以降すべてのコピー・写真・レイアウト判断の基準とする。

### ブランドストーリー（感情訴求版）

```
第一節：記憶
「昔、祖母が仕込んでいた味噌桶の匂いを覚えている。
　寒い朝、湯気の向こうに祖母の背中があった。」

第二節：気づき
「二十四節気という、日本人が千年以上大切にしてきた暦。
　その日その日の"食べる知恵"が、今の暮らしから静かに消えかけている。」

第三節：継承
「九州の小さな味噌蔵で、杜氏が語ってくれた言葉。
　"味噌は、待つことでしか生まれない。"
　その言葉に導かれ、暦とともにめぐる味噌づくりが始まった。」

第四節：約束
「一杯の味噌汁から、今日という季節を感じてほしい。
　それが、暦みそが届けたい暮らしです。」
```

この4節構成を Feature-047（縦タイムラインストーリー型）に流し込み、各節に対応する写真（祖母の記憶＝セピア調の朝食風景、気づき＝暦のクローズアップ、継承＝味噌蔵の杜氏の手元、約束＝現代の食卓）を配置する。

## 5. ロゴ・ブランドアイデンティティ

実際のロゴファイル制作はPhase4のアセット計画（STEP7.5）で改めてChatGPT生成 or ユーザー提供で確定するが、Phase1時点でコンセプトを以下の通り定義する。

### ロゴデザインコンセプト
和文ワードマーク「暦みそ」+ 欧文サブマーク「―KOYOMI―」の2階層構成。
「暦」の一字は、二十四節気を表す小さな二重丸（〇の中に小さな〇）をシンボルマークとして独立させ、単体でもブランドアイコンとして機能させる。

### シンボルマーク
二重丸（同心円）モチーフ＝「暦の巡り」と「味噌桶の断面」を同時に想起させる図形。線は手描き風の墨の滲みタッチ（完全な正円ではなく、わずかな揺らぎを持たせる）。

### ロゴ余白規定
シンボルマーク直径 = X とした場合、上下左右に最低 0.75X のクリアスペースを確保する（他要素・写真端との干渉を防止）。

### ブランドアイコン（セット）
二十四節気を表現する4シーズンの簡易線画アイコン：若葉（春）／水滴（夏）／実り（秋）／雪片（冬）。すべて同心円シンボルと同じ筆致（手描き墨線・単色）で統一する。

### ブランドパターン
同心円シンボルを不規則な間隔・大小でランダム配置した背景パターン（和紙テクスチャに重ねる控えめな透かし用途、opacity 4〜8%）。パッケージ・背景装飾・LINE登録完了画面などに展開可能。

### パッケージイメージ
生成り色のクラフト紙パッケージに、同心円シンボルを一つだけ墨色で刷り、節気名（例：「立春」）を縦書きでラベル表示。過剰な装飾を避け、余白でブランドの静けさを表現する。

## 6. カラー設計（色 + 質感）

単なるカラーコードではなく、色ごとに「質感」を定義する（ブラッシュアップ指示に基づく）。

| 役割 | カラー | 質感定義 |
|---|---|---|
| ベース（生成り） | `#F7F3EA` | 和紙・未晒しの布のような、わずかにザラつきのあるマット質感 |
| ベース（白） | `#FDFCF9` | 障子紙越しの光のような、純白ではなくわずかに温かみのある白 |
| サーフェス（木） | `#B08D5B` / `#8A6A42` | オイル仕上げの木目、経年変化した木のお椀の艶 |
| アクセント（深緑） | `#2F4A3C` | 葉の濃淡、墨を含んだような落ち着いた緑 |
| アクセント（若草） | `#8FA672` | 春の草原、芽吹きの明るさを持つやわらかい緑 |
| アクセント（朱・墨） | `#B5502F`（差し色）/ `#2B2621`（墨） | 味噌の断面の赤み／筆で書いた墨文字の黒 |
| テキスト主 | `#2B2621` | 墨色、純黒ではなく紙に馴染む温かみのある黒 |
| テキスト副 | `#6B5F4F` | 枯れ木色、控えめな説明文用 |

**カラー使用ルール**: 深緑・若草はセクション背景の切り替え用（10〜15%程度の面積）。CTAボタンのみ朱色を単独使用し、視線を集める唯一の高彩度色とする（他は低彩度・低コントラストで統一）。

### 背景の流れ（ブリーフ指定を確定）

```
① Hero: 生成り + 写真
② 共感: 白
③ 薬膳とは: 木目テクスチャ（薄オーバーレイ）
④ ブランドストーリー: 淡いベージュ
⑤ こだわり: 白 + Masonry写真
⑥ 四季の薬膳: 若草 → 深緑（季節ごとにセクション内でグラデーション遷移）
⑦ LINE登録特典: 写真背景（土鍋ご飯・湯気）+ 暗めオーバーレイ
⑧ 愛用者の声: 生成り
⑨ FAQ: 白
⑩ 最終CTA: 生成り + 同心円パターン装飾
```

## 9. 背景デザイン・テクスチャ

単色背景ではなく、以下のテクスチャレイヤーを重ねる。

| テクスチャ | 用途 | 実装方針 |

## 10. レイアウト設計（雑誌構成の原則）

**均等カードグリッドは全セクションで禁止**。以下のルールを徹底する。

```
原則:
- 写真は「大」「中」「小」の最低3サイズを1セクション内で混在させる
- 左右対称配置は避け、セクションごとに軸を左右交互にスイッチする
- 見出し位置は本文カラムの外側（余白側）に配置する箇所を作り、雑誌の柱見出しのような余白の使い方をする
- セクション間の「継ぎ目」を直線区切りではなく、写真のはみ出し・テクスチャのにじみで曖昧にする
- 1カラム全幅の大写真セクション（フルブリード）を2〜3箇所挿入し、スクロールにメリハリを作る
```

### セクション別レイアウト概要（詳細はPhase2ワイヤーフレームで確定）

| # | セクション | レイアウト方針 |
|---|---|---|
① | Hero | 左45%コピー・右55%写真（HERO-008ベース・左右反転）、写真は画面上下フルブリード |
② | 共感 | 中央寄せテキスト、背景白、装飾は最小限（呼吸区間） |
③ | 薬膳とは | FEATURE-015準拠、全幅背景写真+テキストオーバーレイ |

## 11. UIデザイン方針

- ボタン: 角丸だが完全なピル形状ではなく、`border-radius:4px`程度の控えめな角丸（和紙・木の質感に合わせ、通販LPのようなピルボタンは避ける）
- ボーダー: 直線ではなく、要素間の区切りに細い一本線（`1px solid rgba(43,38,33,.12)`程度）を使い、罫線的な和の質感を出す
- アイコン: Lucideではなく、ブランド専用の手描き線画アイコン（節気アイコン4種を基本に、必要に応じて追加）
- フォーム（LINE登録導線）: LPからはLINE公式アカウントの友だち追加ボタン（LINE公式のQR/ボタンAPI）に直接誘導。独自フォームは持たない（ブリーフ「LINE登録が目的」に合わせ導線を単純化）

## 12. アニメーション方針（ブランド体験としての動き）

**派手な演出は禁止**。すべての動きに意味を持たせ、「湯気」「呼吸」「めくり」の3つの動きの語彙に統一する。

| 要素 | 動き | 実装イメージ |
|---|---|---|
| 湯気 | ゆっくり上昇しながら左右にわずかに揺らぐ | SVGパスを`translateY`+`skewX`の複合keyframe、8〜10秒ループ、`ease-in-out` |
| 葉（装飾線画） | わずかな揺れ（既存`sway`パターン応用） | `@keyframes sway{0%,100%{rotate(-2deg)}50%{rotate(2deg)}}`、4〜5秒ループ |
| 背景色の切り替え | セクション間でグラデーションが自然に変化 | セクション背景を`linear-gradient`の多段停止点にし、スクロール連動というより「切り替わりの境界をぼかす」ことで自然遷移を演出 |
| スクロール全体 | 「紙をめくるような印象」 | 各セクションのフェードインを`translateY(24px)→0`+`opacity`のdental-clinic-lp方式（`.reveal-up`, cubic-bezier(.16,.84,.44,1), 0.7s）をベースに、セクション切替時のみわずかな`scale(0.98)→1`を足して「めくり感」を補強 |
| CTA | 少し浮く | `hover: translateY(-3px)`＋影拡大（rapas-lp方式の値を踏襲、`-2px〜-3px`） |
| Hero文字 | 静かなフェードイン（GSAP不使用、素のIntersectionObserverで十分） | opacity 0→1 + translateY(16px)→0、他LPより控えめにdelayを大きめ（0.15s刻み） |

**技術方針**: 本案件はビルドツールなし静的サイトのため、GSAPは導入せず（ai-school-lp/finance-lpのような派手なテック演出は本ブランドの静けさと不整合）、**IntersectionObserver + CSS transitionのみ**で統一する。`prefers-reduced-motion`対応は必須（motion-library.md基準に準拠）。
